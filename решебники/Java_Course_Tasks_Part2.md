## Уровень 21: Точка входа и структура программы (углубленно)

### Теория

**Точка входа в программу** - это место, с которого начинается выполнение программы. В Java это метод `main`.

#### Детальный разбор метода main:

```java
public static void main(String[] args)
```

**Разбор каждого компонента:**

1. **`public`** - модификатор доступа
   - Метод должен быть доступен извне класса
   - JVM (Java Virtual Machine) должна иметь возможность вызвать этот метод
   - Без `public` JVM не сможет найти точку входа

2. **`static`** - принадлежность классу, а не объекту
   - Метод вызывается без создания объекта класса
   - JVM вызывает метод через класс: `ИмяКласса.main(args)`
   - Статические методы существуют до создания объектов

3. **`void`** - отсутствие возвращаемого значения
   - Метод не возвращает результат
   - Программа завершается кодом выхода (exit code)
   - 0 - успешное завершение, не 0 - ошибка

4. **`main`** - имя метода
   - Обязательное имя для точки входа
   - Регистр важен: `Main`, `MAIN` - не сработают
   - Это соглашение, установленное разработчиками Java

5. **`String[] args`** - параметры командной строки
   - Массив строк, переданных при запуске программы
   - Может быть пустым, но параметр обязателен
   - Альтернативные варианты: `String args[]` (работает, но не рекомендуется)

#### Альтернативные сигнатуры main (Java 5+):

```java
// Стандартная (рекомендуется)
public static void main(String[] args)

// Вариативные аргументы (varargs) - работает, но не рекомендуется
public static void main(String... args)

// Обе работают одинаково, но первая более читаема
```

#### Структура Java-программы:

```
Программа Java
│
├── Пакеты (packages) - организация классов
│   └── package com.example;
│
├── Импорты (imports) - подключение библиотек
│   └── import java.util.Scanner;
│
├── Класс (class) - основной контейнер
│   ├── Поля (fields) - данные
│   ├── Методы (methods) - поведение
│   └── Конструкторы (constructors) - инициализация
│
└── Метод main - точка входа
    └── public static void main(String[] args)
```

#### Пример полной структуры:

```java
// Объявление пакета (опционально)
package com.example.program;

// Импорты
import java.util.Scanner;
import java.util.ArrayList;

// Класс
public class МояПрограмма {
    // Статические поля класса
    private static int счетчик = 0;
    
    // Поля экземпляра
    private String имя;
    
    // Статический блок инициализации (выполняется при загрузке класса)
    static {
        System.out.println("Класс загружен в память");
    }
    
    // Конструктор
    public МояПрограмма(String имя) {
        this.имя = имя;
    }
    
    // Точка входа
    public static void main(String[] args) {
        // Код программы
        System.out.println("Программа запущена");
        
        // Обработка аргументов командной строки
        if (args.length > 0) {
            System.out.println("Получено аргументов: " + args.length);
            for (int i = 0; i < args.length; i++) {
                System.out.println("Аргумент " + i + ": " + args[i]);
            }
        }
    }
    
    // Обычные методы
    public void методЭкземпляра() {
        // Работа с полями экземпляра
    }
    
    public static void статическийМетод() {
        // Работа без объекта
    }
}
```

#### Порядок выполнения программы:

1. **Загрузка класса** - JVM загружает класс в память
2. **Инициализация статических полей** - выполняются статические блоки
3. **Поиск метода main** - JVM ищет `public static void main(String[] args)`
4. **Вызов main** - начинается выполнение программы
5. **Выполнение кода** - последовательное выполнение команд
6. **Завершение** - программа завершается (нормально или с ошибкой)

#### Обработка аргументов командной строки:

```java
public static void main(String[] args) {
    // Проверка наличия аргументов
    if (args.length == 0) {
        System.out.println("Аргументы не переданы");
        return; // Досрочный выход
    }
    
    // Обработка аргументов
    for (String arg : args) {
        System.out.println("Аргумент: " + arg);
    }
    
    // Парсинг аргументов
    if (args.length >= 2) {
        String имя = args[0];
        int возраст = Integer.parseInt(args[1]);
        System.out.println("Имя: " + имя + ", Возраст: " + возраст);
    }
}
```

**Запуск с аргументами:**
```bash
java МояПрограмма Никита 20 Москва
# args[0] = "Никита"
# args[1] = "20"
# args[2] = "Москва"
```

## Уровень 22: Компиляция и интерпретация Java-программ

### Теория

Java использует **гибридный подход**: компиляция в байт-код + интерпретация/компиляция в машинный код.

#### Процесс выполнения Java-программы:

```
Исходный код (.java)
    ↓
[Компилятор javac]
    ↓
Байт-код (.class)
    ↓
[JVM - Java Virtual Machine]
    ↓
Машинный код (нативная платформа)
    ↓
Выполнение
```

#### Этап 1: Компиляция (javac)

**Что происходит:**
1. **Лексический анализ** - разбор исходного кода на токены
2. **Синтаксический анализ** - построение дерева синтаксиса
3. **Семантический анализ** - проверка типов, областей видимости
4. **Генерация байт-кода** - создание .class файла

**Команда компиляции:**
```bash
javac ИмяФайла.java
```

**Что создается:**
- Файл `ИмяФайла.class` - байт-код
- Для каждого класса - отдельный .class файл
- Внутренние классы - `Внешний$Внутренний.class`

**Байт-код:**
- Промежуточное представление программы
- Не зависит от платформы
- Выполняется на JVM
- Можно декомпилировать обратно в Java (с потерями)

#### Этап 2: Выполнение (java)

**Что происходит:**
1. **Загрузка класса** - ClassLoader загружает .class файл
2. **Верификация** - проверка безопасности байт-кода
3. **Интерпретация или JIT-компиляция** - выполнение кода
4. **Выполнение** - работа программы

**Команда запуска:**
```bash
java ИмяКласса
```

#### JIT-компиляция (Just-In-Time):

**Как работает:**
- Первые запуски метода - интерпретация байт-кода
- Часто используемые методы компилируются в машинный код
- Машинный код кэшируется для повторного использования
- Оптимизация на основе статистики выполнения

**Преимущества:**
- Быстрое выполнение "горячих" методов
- Адаптивная оптимизация
- Лучше, чем чистая интерпретация

#### Структура .class файла:

```
.class файл
├── Магическое число (0xCAFEBABE)
├── Версия компилятора
├── Константный пул (constant pool)
├── Модификаторы доступа
├── Имя класса
├── Имя суперкласса
├── Интерфейсы
├── Поля
├── Методы
└── Атрибуты
```

#### Просмотр байт-кода:

**Использование javap (Java Disassembler):**
```bash
javap -c ИмяКласса.class
```

**Пример вывода:**
```
Compiled from "Program.java"
public class Program {
  public Program();
    Code:
       0: aload_0
       1: invokespecial #1
       4: return

  public static void main(java.lang.String[]);
    Code:
       0: getstatic     #2
       3: ldc           #3
       5: invokevirtual #4
       8: return
}
```

#### Оптимизации компилятора:

**На этапе компиляции (javac):**
- Удаление неиспользуемого кода
- Константное сворачивание
- Inlining простых методов

**На этапе выполнения (JIT):**
- Inlining методов
- Удаление границ проверок
- Оптимизация циклов
- Специализация методов

#### Сравнение с другими языками:

| Язык | Подход | Скорость компиляции | Скорость выполнения |
|------|--------|---------------------|---------------------|
| C/C++ | Компиляция в машинный код | Медленно | Очень быстро |
| Python | Интерпретация | Нет компиляции | Медленно |
| Java | Компиляция в байт-код + JIT | Средне | Быстро |
| JavaScript | Интерпретация + JIT | Нет компиляции | Средне |

#### Флаги компиляции:

```bash
# Вывод подробной информации
javac -verbose ИмяФайла.java

# Указание версии Java
javac -source 11 -target 11 ИмяФайла.java

# Путь к классам
javac -cp "путь/к/классам" ИмяФайла.java

# Кодировка файлов
javac -encoding UTF-8 ИмяФайла.java
```

#### Флаги выполнения:

```bash
# Указание размера памяти
java -Xmx512m ИмяКласса

# Включение подробного вывода GC
java -XX:+PrintGCDetails ИмяКласса

# Отключение JIT (только интерпретация)
java -Xint ИмяКласса

# Только JIT (без интерпретации)
java -Xcomp ИмяКласса
```

## Уровень 23: Управляющие структуры языка (углубленно)

### Теория

**Управляющие структуры** определяют порядок выполнения инструкций в программе. В Java есть три основных типа: **следование**, **ветвление**, **повторение**.

#### 1. Следование (Sequential Execution)

**Определение:** Выполнение инструкций в порядке их написания, сверху вниз.

**Пример:**
```java
int a = 10;        // Шаг 1
int b = 20;        // Шаг 2
int сумма = a + b; // Шаг 3
System.out.println(сумма); // Шаг 4
```

**Особенности:**
- Каждая инструкция выполняется строго после предыдущей
- Порядок важен
- Нет пропусков или переходов

#### 2. Ветвление (Branching/Conditional)

**Определение:** Выбор одного из нескольких путей выполнения в зависимости от условия.

##### if-else (простое ветвление):

```java
if (условие) {
    // блок 1 - выполнится, если условие true
} else {
    // блок 2 - выполнится, если условие false
}
```

**Внутреннее устройство:**
- Условие вычисляется в boolean значение
- JVM проверяет значение
- Переход к соответствующему блоку кода
- Используются инструкции условных переходов в байт-коде

##### if-else if-else (множественное ветвление):

```java
if (условие1) {
    // блок 1
} else if (условие2) {
    // блок 2
} else if (условие3) {
    // блок 3
} else {
    // блок по умолчанию
}
```

**Важно:** Проверка идет последовательно. Как только найдено первое истинное условие, остальные не проверяются.

##### Тернарный оператор (краткая форма):

```java
результат = (условие) ? значениеЕслиTrue : значениеЕслиFalse;
```

**Пример:**
```java
int max = (a > b) ? a : b;
String статус = (возраст >= 18) ? "Взрослый" : "Ребенок";
```

##### switch-case (множественный выбор):

```java
switch (выражение) {
    case значение1:
        // код для значение1
        break;
    case значение2:
        // код для значение2
        break;
    default:
        // код по умолчанию
}
```

**Особенности:**
- Выражение должно быть целочисленным, char, String или enum
- `break` предотвращает "проваливание" в следующий case
- Без `break` выполнение продолжается дальше (fall-through)

**Switch с выражениями (Java 14+):**
```java
String результат = switch (день) {
    case 1, 2, 3, 4, 5 -> "Рабочий день";
    case 6, 7 -> "Выходной";
    default -> "Неизвестный день";
};
```

#### 3. Повторение (Iteration/Loops)

**Определение:** Многократное выполнение блока кода.

##### Цикл for:

```java
for (инициализация; условие; обновление) {
    // тело цикла
}
```

**Последовательность выполнения:**
1. Инициализация (один раз)
2. Проверка условия
3. Если true → выполнение тела
4. Обновление
5. Возврат к шагу 2

**Расширенный for (for-each):**
```java
for (тип элемент : коллекция) {
    // работа с элементом
}
```

**Внутреннее устройство:**
- Компилятор преобразует for-each в обычный for
- Для массивов - прямой доступ по индексу
- Для коллекций - использование Iterator

##### Цикл while:

```java
while (условие) {
    // тело цикла
}
```

**Особенности:**
- Условие проверяется ПЕРЕД каждой итерацией
- Если условие false с самого начала - цикл не выполнится
- Риск бесконечного цикла, если условие не изменяется

##### Цикл do-while:

```java
do {
    // тело цикла
} while (условие);
```

**Особенности:**
- Тело выполняется минимум один раз
- Условие проверяется ПОСЛЕ выполнения тела
- Полезен, когда нужно гарантировать хотя бы одну итерацию

#### Управление циклами:

##### break - выход из цикла:

```java
for (int i = 0; i < 10; i++) {
    if (i == 5) {
        break; // выходит из цикла при i = 5
    }
    System.out.println(i);
}
// Выведет: 0, 1, 2, 3, 4
```

##### continue - переход к следующей итерации:

```java
for (int i = 0; i < 10; i++) {
    if (i % 2 == 0) {
        continue; // пропускает четные числа
    }
    System.out.println(i);
}
// Выведет: 1, 3, 5, 7, 9
```

##### Метки (labels) - управление вложенными циклами:

```java
внешний: for (int i = 0; i < 3; i++) {
    внутренний: for (int j = 0; j < 3; j++) {
        if (i == 1 && j == 1) {
            break внешний; // выход из внешнего цикла
        }
        System.out.println(i + " " + j);
    }
}
```

#### Оптимизация циклов:

**Хорошие практики:**
```java
// Плохо: вычисление размера на каждой итерации
for (int i = 0; i < список.size(); i++) { }

// Хорошо: сохранение размера
int размер = список.size();
for (int i = 0; i < размер; i++) { }

// Еще лучше: for-each
for (Элемент элемент : список) { }
```

#### Вложенные структуры управления:

**Вложенные if:**
```java
if (условие1) {
    if (условие2) {
        if (условие3) {
            // код
        }
    }
}
```

**Вложенные циклы:**
```java
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        for (int k = 0; k < 3; k++) {
            // код
        }
    }
}
// Сложность: O(n³)
```

## Уровень 24: Простые типы данных (детальное изучение)

### Теория

**Примитивные типы** - это базовые типы данных, встроенные в язык Java. Они хранятся непосредственно в памяти и не являются объектами.

#### Полная таблица примитивных типов:

| Тип | Размер | Диапазон | Значение по умолчанию | Представление в памяти |
|-----|--------|----------|------------------------|------------------------|
| `byte` | 1 байт (8 бит) | -128 до 127 | 0 | Дополнительный код (two's complement) |
| `short` | 2 байта (16 бит) | -32,768 до 32,767 | 0 | Дополнительный код |
| `int` | 4 байта (32 бита) | -2³¹ до 2³¹-1 | 0 | Дополнительный код |
| `long` | 8 байт (64 бита) | -2⁶³ до 2⁶³-1 | 0L | Дополнительный код |
| `float` | 4 байта (32 бита) | ±3.4×10³⁸ | 0.0f | IEEE 754 (single precision) |
| `double` | 8 байт (64 бита) | ±1.7×10³⁰⁸ | 0.0d | IEEE 754 (double precision) |
| `char` | 2 байта (16 бит) | 0 до 65,535 | '\u0000' | UTF-16 (Unicode) |
| `boolean` | ~1 байт* | true/false | false | JVM-зависимо |

*boolean технически не имеет фиксированного размера в спецификации, но обычно занимает 1 байт

#### Представление в памяти:

##### Целочисленные типы (byte, short, int, long):

**Дополнительный код (Two's Complement):**
- Положительные числа: обычное двоичное представление
- Отрицательные числа: инверсия битов + 1

**Пример для byte (8 бит):**
```
0   = 00000000
1   = 00000001
127 = 01111111 (максимум)
-1  = 11111111
-128 = 10000000 (минимум)
```

**Пример для int:**
```java
int число = 42;
// В памяти: 00000000 00000000 00000000 00101010
//            [байт 3][байт 2][байт 1][байт 0]

int отрицательное = -42;
// В памяти: 11111111 11111111 11111111 11010110
```

**Порядок байтов (Endianness):**
- Java использует **big-endian** (старший байт первым)
- Байт 0 - младший (least significant)
- Байт 3 - старший (most significant)

##### Типы с плавающей точкой (float, double):

**IEEE 754 стандарт:**

**Float (32 бита):**
```
[Знак: 1 бит][Порядок: 8 бит][Мантисса: 23 бита]
```

**Double (64 бита):**
```
[Знак: 1 бит][Порядок: 11 бит][Мантисса: 52 бита]
```

**Структура:**
- **Знак**: 0 = положительное, 1 = отрицательное
- **Порядок**: экспонента (смещенная)
- **Мантисса**: дробная часть числа

**Пример:**
```java
float число = 3.14f;
// Представление: приблизительно
// Знак: 0 (положительное)
// Порядок: 128 (представляет степень 2¹)
// Мантисса: дробная часть
```

**Особые значения:**
- `+0.0` и `-0.0` (разные представления, но равны при сравнении)
- `+Infinity` и `-Infinity` (бесконечность)
- `NaN` (Not a Number) - результат недопустимых операций

##### Тип char:

**Unicode и UTF-16:**
- `char` хранит символы Unicode
- Использует UTF-16 кодировку
- Диапазон: U+0000 до U+FFFF (Basic Multilingual Plane)

**Пример:**
```java
char символ = 'A';
// В памяти: 0x0041 (65 в десятичной)

char кириллица = 'Я';
// В памяти: 0x042F (1071 в десятичной)

char эмодзи = '😀'; // Не поддерживается напрямую (нужен String)
```

**Экранированные последовательности:**
```java
'\n' - новая строка (0x000A)
'\t' - табуляция (0x0009)
'\r' - возврат каретки (0x000D)
'\\' - обратный слэш (0x005C)
'\'' - одинарная кавычка (0x0027)
'\"' - двойная кавычка (0x0022)
'\u0041' - Unicode символ (буква A)
```

##### Тип boolean:

**Внутреннее представление:**
- Технически не определено в спецификации
- Обычно 1 байт (0 = false, 1 = true)
- В массивах может быть упакован (1 бит на элемент)
- Нельзя преобразовать в int напрямую

**Ограничения:**
```java
// В Java нельзя:
boolean b = 1;        // Ошибка!
int i = true;         // Ошибка!
if (1) { }            // Ошибка!

// Только так:
boolean b = true;
if (b) { }
```

#### Операции с примитивными типами:

##### Арифметические операции:

```java
int a = 10, b = 3;

a + b  // Сложение: 13
a - b  // Вычитание: 7
a * b  // Умножение: 30
a / b  // Деление: 3 (целочисленное)
a % b  // Остаток: 1
```

**Особенности деления:**
- Для целых типов - целочисленное деление (отбрасывается дробная часть)
- Для float/double - обычное деление
- Деление на ноль: для целых - ArithmeticException, для float/double - Infinity

##### Операции сравнения:

```java
int a = 10, b = 5;

a == b  // Равно: false
a != b  // Не равно: true
a > b   // Больше: true
a < b   // Меньше: false
a >= b  // Больше или равно: true
a <= b  // Меньше или равно: false
```

##### Побитовые операции:

```java
int a = 5;  // 0101 в двоичной
int b = 3;  // 0011 в двоичной

a & b   // Побитовое И: 0001 (1)
a | b   // Побитовое ИЛИ: 0111 (7)
a ^ b   // Побитовое XOR: 0110 (6)
~a      // Побитовое НЕ: ...11111010 (-6)
a << 1  // Сдвиг влево: 1010 (10) - умножение на 2
a >> 1  // Сдвиг вправо: 0010 (2) - деление на 2
a >>> 1 // Беззнаковый сдвиг вправо
```

**Применение:**
- Флаги и маски
- Оптимизация умножения/деления на степени двойки
- Криптография
- Низкоуровневое программирование

##### Операции присваивания:

```java
int a = 10;

a += 5;  // a = a + 5
a -= 3;  // a = a - 3
a *= 2;  // a = a * 2
a /= 2;  // a = a / 2
a %= 3;  // a = a % 3
a &= 7;  // a = a & 7
a |= 2;  // a = a | 2
a ^= 1;  // a = a ^ 1
a <<= 1; // a = a << 1
a >>= 1; // a = a >> 1
```

##### Инкремент и декремент:

```java
int a = 5;

a++;     // Постфиксный инкремент: возвращает 5, затем a = 6
++a;     // Префиксный инкремент: a = 7, затем возвращает 7
a--;     // Постфиксный декремент: возвращает 7, затем a = 6
--a;     // Префиксный декремент: a = 5, затем возвращает 5

// Разница:
int x = 5;
int y = x++;  // y = 5, x = 6

int a = 5;
int b = ++a;  // a = 6, b = 6
```

#### Преобразование типов:

##### Автоматическое преобразование (расширение):

```java
byte → short → int → long → float → double
char → int → long → float → double
```

**Правила:**
- Преобразование происходит автоматически
- Без потери информации (расширение)
- Направление: меньший тип → больший тип

**Примеры:**
```java
byte b = 100;
int i = b;        // Автоматическое расширение

short s = 1000;
long l = s;       // Автоматическое расширение

int i = 100;
double d = i;     // Автоматическое расширение: 100.0
```

##### Явное преобразование (сужение):

```java
double → float → long → int → short → byte
```

**Правила:**
- Требует явного приведения типа
- Может привести к потере информации
- Направление: больший тип → меньший тип

**Примеры:**
```java
double d = 3.14;
int i = (int) d;      // Явное приведение: i = 3 (потеря дробной части)

long l = 1000000;
int i = (int) l;      // OK, если значение помещается

float f = 3.14f;
int i = (int) f;      // i = 3

// Осторожно с переполнением:
int большой = 2000000000;
short s = (short) большой;  // Переполнение! Неожиданное значение
```

**Переполнение:**
```java
byte b = 127;
b++;  // b становится -128 (переполнение)

int i = Integer.MAX_VALUE;  // 2,147,483,647
i++;  // i становится -2,147,483,648 (переполнение)
```

#### Операции с плавающей точкой:

**Точность:**
```java
float f = 0.1f + 0.2f;
System.out.println(f);  // 0.30000001 (неточность!)

double d = 0.1 + 0.2;
System.out.println(d);  // 0.30000000000000004 (неточность!)

// Сравнение:
if (d == 0.3) { }  // false! Не используйте == для float/double

// Правильно:
if (Math.abs(d - 0.3) < 0.0001) { }  // Проверка с допуском
```

**Особые значения:**
```java
double inf = Double.POSITIVE_INFINITY;
double negInf = Double.NEGATIVE_INFINITY;
double nan = Double.NaN;

// Проверки:
Double.isInfinite(inf);     // true
Double.isNaN(nan);          // true
Double.isFinite(100.0);     // true
```

## Уровень 25: Составные типы данных

### Теория

**Составные типы данных** - это типы, которые состоят из нескольких элементов или других типов данных.

#### Массивы (Arrays)

**Определение:** Массив - это структура данных для хранения фиксированного количества элементов одного типа в последовательных ячейках памяти.

##### Представление в памяти:

**Одномерный массив:**
```java
int[] arr = {10, 20, 30, 40, 50};
```

**В памяти:**
```
Адрес:  0x1000  0x1004  0x1008  0x100C  0x1010
Значение:  10     20     30     40     50
Индекс:    0      1      2      3      4
```

**Особенности:**
- Элементы расположены последовательно
- Размер фиксирован после создания
- Доступ по индексу: O(1) - константное время
- Размер массива: `arr.length` (не метод!)

**Многомерные массивы:**

**Двумерный массив (матрица):**
```java
int[][] matrix = {
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9}
};
```

**В памяти (реализация как массив массивов):**
```
matrix[0] → [1, 2, 3]
matrix[1] → [4, 5, 6]
matrix[2] → [7, 8, 9]
```

**Размеры:**
```java
matrix.length        // Количество строк: 3
matrix[0].length     // Количество столбцов: 3
```

**Неровные массивы (jagged arrays):**
```java
int[][] jagged = {
    {1, 2},
    {3, 4, 5, 6},
    {7}
};
// Каждая строка может иметь разную длину
```

##### Операции с массивами:

**Создание:**
```java
// Способ 1: с начальными значениями
int[] arr1 = {1, 2, 3, 4, 5};

// Способ 2: с указанием размера
int[] arr2 = new int[5];  // Все элементы = 0

// Способ 3: объявление и инициализация
int[] arr3 = new int[]{1, 2, 3};
```

**Доступ к элементам:**
```java
int[] arr = {10, 20, 30};
int первый = arr[0];      // 10
int последний = arr[arr.length - 1];  // 30

// Изменение
arr[1] = 25;  // Теперь: {10, 25, 30}
```

**Копирование:**
```java
int[] original = {1, 2, 3, 4, 5};

// Поверхностное копирование (только ссылка)
int[] copy1 = original;

// Глубокое копирование
int[] copy2 = Arrays.copyOf(original, original.length);
int[] copy3 = original.clone();

// Копирование части массива
int[] copy4 = Arrays.copyOfRange(original, 1, 4);  // {2, 3, 4}
```

**Сравнение:**
```java
int[] arr1 = {1, 2, 3};
int[] arr2 = {1, 2, 3};

arr1 == arr2;              // false (сравнение ссылок)
Arrays.equals(arr1, arr2); // true (сравнение содержимого)
```

#### Строки (String)

**Определение:** String - это класс для работы с последовательностью символов. В Java строки **неизменяемы** (immutable).

##### Представление в памяти:

**Структура String:**
```java
String str = "Hello";
```

**В памяти:**
```
Объект String:
├── char[] value = {'H', 'e', 'l', 'l', 'o'}
├── int offset = 0
├── int count = 5
└── int hash = (кэшированное значение)
```

**String Pool (пул строк):**
```java
String s1 = "Hello";      // Создается в String Pool
String s2 = "Hello";      // Используется из String Pool
String s3 = new String("Hello");  // Новый объект вне Pool

s1 == s2;  // true (одна и та же ссылка)
s1 == s3;  // false (разные объекты)
s1.equals(s3);  // true (одинаковое содержимое)
```

**Intern (добавление в Pool):**
```java
String s1 = new String("Hello");
String s2 = s1.intern();  // Добавляет в Pool или возвращает существующую
String s3 = "Hello";
s2 == s3;  // true
```

##### Операции со строками:

**Создание:**
```java
String s1 = "Hello";
String s2 = new String("Hello");
String s3 = new String(new char[]{'H', 'e', 'l', 'l', 'o'});
```

**Конкатенация:**
```java
String s1 = "Hello";
String s2 = "World";
String s3 = s1 + " " + s2;  // "Hello World"

// Внутри: создается StringBuilder, затем toString()
```

**Сравнение:**
```java
String s1 = "Hello";
String s2 = "hello";

s1.equals(s2);           // false (регистр важен)
s1.equalsIgnoreCase(s2); // true
s1.compareTo(s2);        // Отрицательное число (лексикографическое)
s1.compareToIgnoreCase(s2); // 0
```

#### Классы (Classes)

**Определение:** Класс - это шаблон для создания объектов, определяющий структуру данных и поведение.

##### Представление объекта в памяти:

**Структура объекта:**
```java
class Person {
    String name;
    int age;
}

Person p = new Person();
```

**В памяти (Heap):**
```
Объект Person:
├── Заголовок объекта (header)
│   ├── Mark Word (информация для GC)
│   ├── Класс-указатель (ссылка на Class)
│   └── Размер массива (если массив)
├── name (ссылка на String) → String объект в Heap
└── age (int = 0)
```

**Размер объекта:**
- Заголовок: 8-16 байт (зависит от JVM, 64-bit)
- Поля: размер зависит от типа
- Выравнивание: объекты выравниваются по 8 байт

**Пример расчета:**
```java
class Example {
    byte b;      // 1 байт
    int i;       // 4 байта
    long l;      // 8 байт
    String s;    // 4/8 байт (ссылка)
}
// Общий размер: ~32-40 байт (с учетом выравнивания)
```

##### Ссылки и объекты:

**Ссылочные типы:**
```java
Person p1 = new Person();
Person p2 = p1;  // p2 указывает на тот же объект

p1.name = "Иван";
System.out.println(p2.name);  // "Иван" (тот же объект!)
```

**В памяти:**
```
Стек (Stack):
p1 → [ссылка] → 0x1000
p2 → [ссылка] → 0x1000 (та же ссылка!)

Куча (Heap):
0x1000: Person объект
```

**Сравнение ссылок:**
```java
Person p1 = new Person();
Person p2 = new Person();
Person p3 = p1;

p1 == p2;  // false (разные объекты)
p1 == p3;  // true (одна ссылка)
p1.equals(p2);  // зависит от реализации equals()
```

#### Коллекции (Collections)

**ArrayList - динамический массив:**

**Внутренняя структура:**
```java
ArrayList<Integer> list = new ArrayList<>();
```

**В памяти:**
```
ArrayList объект:
├── Object[] elementData (внутренний массив)
├── int size (количество элементов)
└── int modCount (для fail-fast итераторов)

elementData: [1, 2, 3, null, null, null, ...]
size: 3
capacity: 10 (по умолчанию)
```

**Операции:**
- `add()` - O(1) амортизированно, O(n) при расширении
- `get()` - O(1)
- `remove()` - O(n) (сдвиг элементов)
- `contains()` - O(n)

**HashMap - хеш-таблица:**

**Внутренняя структура:**
```java
HashMap<String, Integer> map = new HashMap<>();
```

**В памяти (Java 8+):**
```
HashMap объект:
├── Node[] table (массив бакетов)
├── int size
├── int threshold
└── float loadFactor

table[0] → [ключ1=значение1] → [ключ2=значение2] (цепочка)
table[1] → null
table[2] → [ключ3=значение3]
...
```

**Хеширование:**
```java
String key = "Hello";
int hashCode = key.hashCode();  // Вычисление хеш-кода
int index = hashCode % table.length;  // Индекс бакета
```

**Операции:**
- `put()` - O(1) в среднем, O(n) в худшем случае
- `get()` - O(1) в среднем, O(n) в худшем случае
- `containsKey()` - O(1) в среднем

## Уровень 26: Организация ввода-вывода (stdin-stdout)

### Теория

**Ввод-вывод (I/O)** - это механизм обмена данными между программой и внешним миром (консоль, файлы, сеть).

#### Стандартные потоки:

**Три основных потока:**
1. **stdin (Standard Input)** - стандартный ввод (клавиатура)
2. **stdout (Standard Output)** - стандартный вывод (консоль)
3. **stderr (Standard Error)** - стандартный поток ошибок

**В Java:**
```java
System.in   // InputStream - стандартный ввод
System.out  // PrintStream - стандартный вывод
System.err  // PrintStream - стандартный поток ошибок
```

#### Вывод (stdout):

##### System.out.println():

**Что происходит:**
```java
System.out.println("Hello");
```

**Процесс:**
1. Строка "Hello" создается в памяти
2. Вызывается метод `println()` объекта `PrintStream`
3. Данные буферизуются
4. Записываются в консоль через системный вызов
5. Переход на новую строку

**Варианты вывода:**
```java
System.out.print("Текст");        // Без перевода строки
System.out.println("Текст");      // С переводом строки
System.out.printf("Формат", args); // Форматированный вывод
```

##### Форматированный вывод (printf):

**Синтаксис:**
```java
System.out.printf("Формат", аргумент1, аргумент2, ...);
```

**Спецификаторы формата:**
```java
%d  // Десятичное целое число
%f  // Число с плавающей точкой
%s  // Строка
%c  // Символ
%b  // Boolean
%x  // Шестнадцатеричное число
%o  // Восьмеричное число
%e  // Научная нотация
%n  // Перевод строки (платформо-независимый)
```

**Примеры:**
```java
int число = 42;
double цена = 99.99;
String имя = "Никита";

System.out.printf("Число: %d%n", число);
// Вывод: Число: 42

System.out.printf("Цена: %.2f рублей%n", цена);
// Вывод: Цена: 99.99 рублей

System.out.printf("Имя: %s, Возраст: %d%n", имя, 20);
// Вывод: Имя: Никита, Возраст: 20

System.out.printf("Шестнадцатеричное: %x%n", 255);

```

**Флаги форматирования:**

## Уровень 27: Управление памятью и сборщик мусора

### Теория

**Управление памятью** - это процесс выделения и освобождения памяти для объектов программы. Java автоматически управляет памятью через сборщик мусора (Garbage Collector).

#### Типы памяти в Java:

##### 1. Стек (Stack Memory)

**Назначение:**
- Хранит локальные переменные и ссылки на объекты
- Хранит вызовы методов (стек вызовов)
- Быстрый доступ
- Ограниченный размер

**Что хранится:**
```java
public void метод() {
    int локальнаяПеременная = 10;  // В стеке
    String ссылка;                  // В стеке (сама ссылка)
    // Объект String находится в куче!
}
```

**Структура стека:**
```
Стек вызовов:
┌─────────────────┐
│ main()          │ ← Вершина стека
│   переменные    │
├─────────────────┤
│ метод1()        │
│   переменные    │
├─────────────────┤
│ метод2()        │
│   переменные    │
└─────────────────┘
```

**Особенности:**
- LIFO (Last In, First Out) - последний вошел, первый вышел
- Автоматическое управление
- Ограниченный размер (обычно 1-2 МБ)
- StackOverflowError при переполнении

**Размер стека:**
```bash
# Увеличение размера стека
java -Xss2m ИмяКласса  # 2 МБ
```

##### 2. Куча (Heap Memory)

**Назначение:**
- Хранит все объекты
- Динамическое выделение памяти
- Управляется сборщиком мусора

**Структура кучи:**
```
Куча (Heap)
├── Young Generation (Молодое поколение)
│   ├── Eden Space (Райский сад)
│   └── Survivor Spaces (S0, S1)
│
└── Old Generation (Старое поколение)
    └── Tenured Space
```

**Разделение на поколения:**

**Young Generation:**
- Новые объекты создаются здесь
- Быстрая сборка мусора (Minor GC)
- Небольшой размер
- Большинство объектов умирает быстро

**Old Generation:**
- Объекты, пережившие несколько сборок
- Медленная сборка (Major GC / Full GC)
- Большой размер
- Долгоживущие объекты

**Размеры кучи:**
```bash
# Установка начального размера
java -Xms512m ИмяКласса

# Установка максимального размера
java -Xmx2g ИмяКласса

# Оба параметра
java -Xms512m -Xmx2g ИмяКласса
```

##### 3. Метасpace (Metaspace) / PermGen (до Java 8)

**Назначение:**
- Хранит метаданные классов
- Информация о классах, методах, полях
- Не является частью кучи

**PermGen (до Java 8):**
- Ограниченный размер
- OutOfMemoryError: PermGen space

**Metaspace (Java 8+):**
- Автоматическое управление размером
- Использует нативную память
- OutOfMemoryError: Metaspace

**Размер Metaspace:**
```bash
java -XX:MetaspaceSize=256m ИмяКласса
java -XX:MaxMetaspaceSize=512m ИмяКласса
```

#### Статическая память:

**Статические переменные:**
```java
class Example {
    static int статическаяПеременная = 10;  // В Metaspace
    static String статическаяСтрока = "Hello";
}
```

**Особенности:**
- Существует на протяжении всей жизни программы
- Одна копия на весь класс (не на объект)
- Инициализируется при загрузке класса
- Хранится в Metaspace

**Статические блоки:**
```java
class Example {
    static {
        // Выполняется один раз при загрузке класса
        System.out.println("Класс загружен");
    }
}
```

#### Динамическая память (Heap):

**Выделение памяти:**
```java
// Создание объекта - выделение памяти в куче
Person person = new Person();
```

**Процесс:**
1. JVM проверяет наличие свободного места в Eden Space
2. Выделяет память для объекта
3. Инициализирует объект (конструктор)
4. Возвращает ссылку

**Освобождение памяти:**
- Автоматическое через сборщик мусора
- Программист не управляет напрямую
- Можно предложить сборку: `System.gc()` (не гарантирует сборку!)

#### Сборщик мусора (Garbage Collector):

**Что такое мусор:**
- Объекты, на которые нет ссылок
- Объекты, недостижимые из корневых ссылок

**Корневые ссылки (GC Roots):**
- Локальные переменные в активных методах
- Статические переменные
- Ссылки в нативных методах
- Ссылки в потоках

**Алгоритм работы:**

**1. Mark (Пометка):**
```
Начинаем с GC Roots
    ↓
Помечаем все достижимые объекты
    ↓
Рекурсивно помечаем объекты, на которые они ссылаются
```

**2. Sweep (Очистка):**
```
Проходим по всей куче
    ↓
Удаляем непомеченные объекты
    ↓
Освобождаем память
```

**3. Compact (Уплотнение) - опционально:**
```
Перемещаем живые объекты
    ↓
Освобождаем непрерывные блоки памяти
```

**Типы сборщиков мусора:**

**1. Serial GC:**
- Однопоточный
- Простой алгоритм
- Подходит для небольших приложений
- Флаг: `-XX:+UseSerialGC`

**2. Parallel GC:**
- Многопоточный
- Быстрее для больших приложений
- Флаг: `-XX:+UseParallelGC`

**3. G1 GC (Garbage First):**
- Разделяет кучу на регионы
- Собирает регионы с наибольшим количеством мусора
- Низкие паузы
- Флаг: `-XX:+UseG1GC`

**4. ZGC / Shenandoah:**
- Очень низкие паузы (< 10ms)
- Для больших куч (TB)
- Экспериментальные

**Minor GC vs Major GC:**

**Minor GC:**
- Сборка в Young Generation
- Быстрая (миллисекунды)
- Частая
- Не останавливает приложение надолго

**Major GC (Full GC):**
- Сборка во всей куче
- Медленная (секунды)
- Редкая
- Останавливает приложение (Stop-The-World)

**Мониторинг GC:**
```bash
# Вывод информации о GC
java -XX:+PrintGCDetails ИмяКласса

# Вывод времени GC
java -XX:+PrintGCTimeStamps ИмяКласса

# Логирование в файл
java -Xlog:gc:gc.log ИмяКласса
```

#### Утечки памяти (Memory Leaks):

**Что это:**
- Объекты не освобождаются, хотя не используются
- Память постепенно заполняется
- OutOfMemoryError в итоге

**Типичные причины:**

**1. Неиспользуемые ссылки:**
```java
// Плохо
List<Object> список = new ArrayList<>();
// ... использование
список = null; // Но объекты в списке все еще ссылаются друг на друга
```

**2. Статические коллекции:**
```java
class Cache {
    private static Map<String, Object> cache = new HashMap<>();
    // Объекты никогда не удаляются!
}
```

**3. Слушатели событий:**
```java
// Не удален слушатель - объект не может быть собран
button.addActionListener(listener);
// Нужно: button.removeActionListener(listener);
```

**4. Внутренние классы:**
```java
class Outer {
    class Inner {
        // Inner хранит ссылку на Outer
    }
}
```

**Как избежать:**
- Использовать WeakReference для кэшей
- Очищать коллекции
- Удалять слушатели
- Использовать try-with-resources

#### WeakReference и SoftReference:

**Strong Reference (обычная ссылка):**
```java
Object obj = new Object();  // Объект не будет собран, пока есть ссылка
```

**WeakReference:**
```java
WeakReference<Object> weakRef = new WeakReference<>(new Object());
Object obj = weakRef.get();  // Может быть null, если объект собран
// Объект может быть собран, если нет других ссылок
```

**SoftReference:**
```java
SoftReference<Object> softRef = new SoftReference<>(new Object());
// Объект соберется только при нехватке памяти
```

**Применение:**
- Кэширование
- Избежание утечек памяти
- Слабые связи между объектами

#### Оптимизация использования памяти:

**1. Переиспользование объектов:**
```java
// Плохо: создание новых объектов
for (int i = 0; i < 1000; i++) {
    String s = new String("Hello");
}

// Хорошо: переиспользование
String s = "Hello";
for (int i = 0; i < 1000; i++) {
    // Используем s
}
```

**2. Избежание упаковки (boxing):**
```java
// Плохо: создание объектов Integer
List<Integer> list = new ArrayList<>();
for (int i = 0; i < 1000000; i++) {
    list.add(i);  // Автоупаковка: new Integer(i)
}

// Хорошо: использование примитивов где возможно
int[] arr = new int[1000000];
```

**3. Очистка больших коллекций:**
```java
List<Object> большойСписок = new ArrayList<>();
// ... использование
большойСписок.clear();  // Освобождает ссылки
большойСписок = null;   // Помогает GC
```

## Уровень 28: Дженерики (Generics) и сложные структуры данных

### Теория

**Дженерики (Generics)** - это механизм параметризации типов, позволяющий создавать классы, интерфейсы и методы, работающие с разными типами данных.

#### Зачем нужны дженерики:

**Проблема без дженериков:**
```java
// Старый способ (до Java 5)
List список = new ArrayList();
список.add("Строка");
список.add(10);  // Ошибка обнаружится только во время выполнения!
String s = (String) список.get(1);  // ClassCastException!
```

**Решение с дженериками:**
```java
// Новый способ
List<String> список = new ArrayList<>();
список.add("Строка");
// список.add(10);  // Ошибка компиляции!
String s = список.get(0);  // Безопасно, без приведения типов
```

**Преимущества:**
- Безопасность типов на этапе компиляции
- Устранение необходимости приведения типов
- Лучшая читаемость кода
- Переиспользование кода

#### Синтаксис дженериков:

##### Объявление generic класса:

```java
public class Box<T> {
    private T содержимое;
    
    public void положить(T предмет) {
        this.содержимое = предмет;
    }
    
    public T взять() {
        return содержимое;
    }
}
```

**Использование:**
```java
Box<String> коробкаСтрок = new Box<>();
коробкаСтрок.положить("Hello");
String строка = коробкаСтрок.взять();  // Без приведения типов

Box<Integer> коробкаЧисел = new Box<>();
коробкаЧисел.положить(42);
int число = коробкаЧисел.взять();  // Автоупаковка/распаковка
```

##### Несколько параметров типа:

```java
public class Pair<K, V> {
    private K ключ;
    private V значение;
    
    public Pair(K ключ, V значение) {
        this.ключ = ключ;
        this.значение = значение;
    }
    
    public K получитьКлюч() { return ключ; }
    public V получитьЗначение() { return значение; }
}
```

**Использование:**
```java
Pair<String, Integer> пара = new Pair<>("Возраст", 25);
String ключ = пара.получитьКлюч();
int значение = пара.получитьЗначение();
```

##### Ограничения типов (Bounded Type Parameters):

**Верхняя граница (extends):**
```java
public class NumberBox<T extends Number> {
    private T число;
    
    public double получитьDouble() {
        return число.doubleValue();  // Метод Number доступен
    }
}

// Использование:
NumberBox<Integer> коробка1 = new NumberBox<>();  // OK
NumberBox<Double> коробка2 = new NumberBox<>();  // OK
// NumberBox<String> коробка3 = new NumberBox<>();  // Ошибка!
```

**Несколько границ:**
```java
public class Example<T extends Number & Comparable<T>> {
    // T должен быть Number И реализовывать Comparable
}
```

**Нижняя граница (super) - в wildcards:**
```java
public void метод(List<? super Integer> список) {
    // Принимает List<Integer>, List<Number>, List<Object>
}
```

##### Wildcards (Подстановочные знаки):

**? (неограниченный wildcard):**
```java
public void вывестиСписок(List<?> список) {
    // Принимает List любого типа
    // Но нельзя добавлять элементы (кроме null)
    for (Object obj : список) {
        System.out.println(obj);
    }
}
```

**? extends T (верхняя граница):**
```java
public void вывестиЧисла(List<? extends Number> список) {
    // Принимает List<Number>, List<Integer>, List<Double>
    // Можно читать как Number
    // Нельзя добавлять (кроме null)
    for (Number n : список) {
        System.out.println(n.doubleValue());
    }
}
```

**? super T (нижняя граница):**
```java
public void добавитьЧисла(List<? super Integer> список) {
    // Принимает List<Integer>, List<Number>, List<Object>
    // Можно добавлять Integer и его подтипы
    список.add(10);
    список.add(20);
    // Можно читать как Object
}
```

**PECS принцип (Producer Extends, Consumer Super):**
- Если коллекция производит элементы → используйте `extends`
- Если коллекция потребляет элементы → используйте `super`

#### Дженерики в методах:

**Generic методы:**
```java
public class Утилиты {
    // Generic метод в обычном классе
    public static <T> T получитьПервый(List<T> список) {
        if (список.isEmpty()) {
            return null;
        }
        return список.get(0);
    }
    
    // Несколько параметров типа
    public static <T, U> U преобразовать(T элемент, Function<T, U> функция) {
        return функция.apply(элемент);
    }
}
```

**Использование:**
```java
List<String> строки = Arrays.asList("один", "два", "три");
String первый = Утилиты.<String>получитьПервый(строки);
// Или с автоматическим выводом типа:
String первый = Утилиты.получитьПервый(строки);
```

#### Стирание типов (Type Erasure):

**Что происходит:**
- На этапе компиляции дженерики проверяются
- На этапе выполнения информация о типах стирается
- Все дженерики становятся Object (или верхней границей)

**До компиляции:**
```java
List<String> список = new ArrayList<>();
```

**После компиляции (type erasure):**
```java
List список = new ArrayList();  // T заменен на Object
```

**Последствия:**
```java
// Нельзя:
if (obj instanceof List<String>) { }  // Ошибка компиляции

// Можно:
if (obj instanceof List) { }  // OK

// Нельзя создать массив:
List<String>[] массив = new List<String>[10];  // Ошибка!

// Можно:
List<String>[] массив = new List[10];  // С предупреждением
```

##### Generic Stack (Стек):

```java
public class Stack<T> {
    private List<T> элементы = new ArrayList<>();
    
    public void push(T элемент) {
        элементы.add(элемент);
    }
    
    public T pop() {
        if (элементы.isEmpty()) {
            throw new EmptyStackException();
        }
        return элементы.remove(элементы.size() - 1);
    }
    
    public T peek() {
        if (элементы.isEmpty()) {
            throw new EmptyStackException();
        }
        return элементы.get(элементы.size() - 1);
    }
    
    public boolean isEmpty() {
        return элементы.isEmpty();
    }
    
    public int size() {
        return элементы.size();
    }
}
```

**Использование:**
```java
Stack<Integer> стек = new Stack<>();
стек.push(10);
стек.push(20);
стек.push(30);
System.out.println(стек.pop());  // 30
System.out.println(стек.peek()); // 20
```

##### Generic Binary Tree (Бинарное дерево):

```java
public class BinaryTree<T extends Comparable<T>> {
    private class Node {
        T значение;
        Node левый;
        Node правый;
        
        Node(T значение) {
            this.значение = значение;
        }
    }
    
    private Node корень;
    
    public void добавить(T значение) {
        корень = добавитьРекурсивно(корень, значение);
    }
    
    private Node добавитьРекурсивно(Node узел, T значение) {
        if (узел == null) {
            return new Node(значение);
        }
        
        if (значение.compareTo(узел.значение) < 0) {
            узел.левый = добавитьРекурсивно(узел.левый, значение);
        } else if (значение.compareTo(узел.значение) > 0) {
            узел.правый = добавитьРекурсивно(узел.правый, значение);
        }
        
        return узел;
    }
    
    public boolean содержит(T значение) {
        return содержитРекурсивно(корень, значение);
    }
    
    private boolean содержитРекурсивно(Node узел, T значение) {
        if (узел == null) {
            return false;
        }
        
        int сравнение = значение.compareTo(узел.значение);
        if (сравнение == 0) {
            return true;
        } else if (сравнение < 0) {
            return содержитРекурсивно(узел.левый, значение);
        } else {
            return содержитРекурсивно(узел.правый, значение);
        }
    }
}
```

##### Generic HashMap (Хеш-таблица):

```java
public class HashMap<K, V> {
    private static final int НАЧАЛЬНЫЙ_РАЗМЕР = 16;
    private List<Entry<K, V>>[] бакеты;
    
    private static class Entry<K, V> {
        K ключ;
        V значение;
        Entry<K, V> следующий;
        
        Entry(K ключ, V значение) {
            this.ключ = ключ;
            this.значение = значение;
        }
    }
    
    @SuppressWarnings("unchecked")
    public HashMap() {
        бакеты = new List[НАЧАЛЬНЫЙ_РАЗМЕР];
        for (int i = 0; i < бакеты.length; i++) {
            бакеты[i] = new ArrayList<>();
        }
    }
    
    public void put(K ключ, V значение) {
        int индекс = хеш(ключ) % бакеты.length;
        List<Entry<K, V>> бакет = бакеты[индекс];
        
        for (Entry<K, V> entry : бакет) {
            if (entry.ключ.equals(ключ)) {
                entry.значение = значение;
                return;
            }
        }
        
        бакет.add(new Entry<>(ключ, значение));
    }
    
    public V get(K ключ) {
        int индекс = хеш(ключ) % бакеты.length;
        List<Entry<K, V>> бакет = бакеты[индекс];
        
        for (Entry<K, V> entry : бакет) {
            if (entry.ключ.equals(ключ)) {
                return entry.значение;
            }
        }
        
        return null;
    }
    
    private int хеш(K ключ) {
        return ключ.hashCode();
    }
}
```
