# Курс Java для начинающих - Теория и Задачи

## Введение

Java - это объектно-ориентированный язык программирования высокого уровня, разработанный компанией Sun Microsystems (теперь Oracle). Java популярен благодаря своей кроссплатформенности - программа, написанная на Java, может работать на любой операционной системе.

**Особенности Java:**
- Объектно-ориентированный подход
- Автоматическое управление памятью (сборка мусора)
- Кроссплатформенность (Write Once, Run Anywhere)
- Безопасность и надежность
- Большая стандартная библиотека

**Структура простой программы:**
```java
public class ИмяКласса {
    public static void main(String[] args) {
        // Ваш код здесь
        System.out.println("Привет, мир!");
    }
}
```

**Разбор структуры:**
- `public class ИмяКласса` - объявление класса. `public` означает, что класс доступен извне. `class` - ключевое слово для создания класса.
- `public static void main(String[] args)` - метод `main`, точка входа в программу:
  - `public` - метод доступен извне
  - `static` - метод принадлежит классу, а не объекту
  - `void` - метод ничего не возвращает
  - `main` - имя метода (обязательно для запуска программы)
  - `String[] args` - параметры командной строки (пока не используем)
- `System.out.println()` - выводит текст на экран и переходит на новую строку
- `//` - комментарий, игнорируется компилятором

**Важно:**
- Имя класса должно совпадать с именем файла (например, класс `HelloWorld` в файле `HelloWorld.java`)
- Java чувствителен к регистру (HelloWorld ≠ helloworld)
- Каждая команда заканчивается точкой с запятой `;`
- Фигурные скобки `{}` обозначают блоки кода

**Процесс работы с Java:**

**Способ 1: Классический (работает всегда):**
1. Пишете код в файле `.java`
2. Компилируете: `javac ИмяФайла.java` (создается `.class` файл)
3. Запускаете: `java ИмяКласса` (без расширения `.java` и `.class`)

**Способ 2: Прямой запуск (Java 11 и выше):**
1. Пишете код в файле `.java`
2. Запускаете напрямую: `java ИмяФайла.java` (компилирует и запускает автоматически)

**Что значит "один класс в файле"?**

Класс - это блок кода, который начинается с `public class ИмяКласса { ... }`.

**✅ Работает с прямым запуском (один класс):**
```java
// Файл: HelloWorld.java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Привет!");
    }
}
// Только ОДИН класс HelloWorld - можно запустить: java HelloWorld.java
```

**❌ НЕ работает с прямым запуском (несколько классов):**
```java
// Файл: Program.java
public class Program {
    // main ТОЛЬКО здесь - это точка входа в программу
    public static void main(String[] args) {
        Человек человек = new Человек();  // используем второй класс
        человек.имя = "Никита";
        человек.представиться();
    }
}

class Человек {  // ВТОРОЙ класс - вспомогательный, main НЕ нужен!
    String имя;
    void представиться() {
        System.out.println("Привет, я " + имя);
    }
}
// ДВА класса в одном файле - нужно использовать: javac Program.java, затем java Program
```

**Важно:** 
- `main` нужен только в ОДНОМ классе (главном, который запускается)
- Остальные классы - это вспомогательные "инструменты", они не запускаются сами по себе
- Java запускает программу с класса, где есть `main`, а остальные классы используются по необходимости

**Важно:** Прямой запуск `java имя_файла.java` работает только для простых программ с одним классом. Для сложных проектов (несколько классов, пакеты) используйте классический способ.

## Уровень 1: Основы (Переменные и вывод)

### Теория

**Переменные** - это именованные контейнеры для хранения данных в памяти компьютера. Переменная имеет имя, тип и значение. В Java нужно обязательно указать тип переменной перед её именем - это называется статической типизацией.

**Основные типы переменных:**
- `int` - целые числа (например: 5, -10, 100, 0). Диапазон: от -2,147,483,648 до 2,147,483,647
- `double` - десятичные числа с плавающей точкой (например: 3.14, 2.5, -0.5). Точность до 15-17 знаков
- `String` - текст (например: "Привет", "Java", "123"). Обратите внимание на двойные кавычки
- `boolean` - логические значения (только `true` или `false`)

**Синтаксис создания переменной:**
```java
тип имяПеременной = значение;
```

**Правила именования переменных:**
- Имя должно начинаться с буквы, подчеркивания `_` или знака доллара `$`
- Может содержать буквы, цифры, `_` и `$`
- Не может быть ключевым словом Java (if, for, class и т.д.)
- Java чувствителен к регистру (age ≠ Age)
- Рекомендуется использовать понятные имена (age вместо a)

**Примеры:**
```java
int возраст = 20;              // целое число
String имя = "Никита";         // текст
double цена = 99.99;           // десятичное число
boolean студент = true;        // логическое значение
```

**Изменение значения переменной:**
```java
int число = 10;
число = 20;  // теперь число = 20
число = число + 5;  // теперь число = 25
```

**Вывод на экран:**
```java
System.out.println("Текст");  // Выводит текст и переходит на новую строку
System.out.print("Текст");    // Выводит текст без перехода на новую строку
```

**Разница между println и print:**
```java
System.out.print("Привет");
System.out.print("мир");
// Выведет: Приветмир (в одну строку)

System.out.println("Привет");
System.out.println("мир");
// Выведет:
// Привет
// мир
// (в две строки)
```

**Вывод переменных:**
```java
int возраст = 20;
String имя = "Никита";
System.out.println("Меня зовут " + имя + ", мне " + возраст + " лет");
// Выведет: Меня зовут Никита, мне 20 лет
```

**Арифметические операции:**
- `+` - сложение (также для объединения строк)
- `-` - вычитание
- `*` - умножение
- `/` - деление (для целых чисел - целочисленное деление)
- `%` - остаток от деления (модуло)

**Примеры операций:**
```java
int a = 10;
int b = 5;
int сумма = a + b;        // 15
int разность = a - b;     // 5
int произведение = a * b; // 50
int частное = a / b;      // 2 (целочисленное деление)
int остаток = a % b;      // 0 (10 делится на 5 без остатка)

int c = 7;
int d = 3;
int частное2 = c / d;     // 2 (не 2.33, так как int)
int остаток2 = c % d;     // 1 (7 = 2*3 + 1)
```

**Операции с double:**
```java
double x = 10.0;
double y = 3.0;
double результат = x / y;  // 3.3333333333333335 (точное деление)
```

**Сокращенные операции:**
```java
int число = 10;
число += 5;  // то же что: число = число + 5
число -= 3;  // то же что: число = число - 3
число *= 2;  // то же что: число = число * 2
число /= 2;  // то же что: число = число / 2
число++;     // то же что: число = число + 1 (инкремент)
число--;     // то же что: число = число - 1 (декремент)
```

## Уровень 2: Типы данных

### Теория

В Java все данные имеют определенный тип. Тип определяет, какие значения может хранить переменная и какие операции с ней можно выполнять.

**Примитивные типы** - это базовые типы данных, встроенные в язык:

**Целочисленные типы:**
- `byte` - от -128 до 127 (1 байт = 8 бит). Используется редко, для экономии памяти
- `short` - от -32,768 до 32,767 (2 байта). Используется редко
- `int` - от -2,147,483,648 до 2,147,483,647 (4 байта) - **самый используемый**
- `long` - от -9,223,372,036,854,775,808 до 9,223,372,036,854,775,807 (8 байт). Для очень больших чисел, нужно добавлять `L`: `long большое = 10000000000L;`

**Числа с плавающей точкой:**
- `float` - 32 бита, точность ~7 знаков после запятой. Нужно добавлять `f`: `float число = 3.14f;`
- `double` - 64 бита, точность ~15-17 знаков после запятой - **самый используемый**

**Символы:**
- `char` - один символ Unicode (2 байта). Использует одинарные кавычки: `'A'`, `'я'`, `'5'`, `' '` (пробел)

**Логический тип:**
- `boolean` - только `true` или `false` (1 байт). В Java нельзя использовать числа вместо boolean

**Строки:**
- `String` - это не примитивный тип, а класс, но используется очень часто. Строка - это последовательность символов. Использует двойные кавычки: `"Привет"`, `"Java"`, `"123"` (это текст, не число!)

**Примеры:**
```java
int число = 42;
double десятичное = 3.14;
boolean правда = true;
boolean ложь = false;
char символ = 'A';
String текст = "Привет";
```

**Важные различия:**
```java
char символ = 'A';        // один символ, одинарные кавычки
String строка = "A";      // строка из одного символа, двойные кавычки

int число = 5;            // число 5
String текст = "5";       // текст "5" (не число!)
```

**Преобразование типов:**
```java
// Автоматическое преобразование (расширение)
int маленькое = 10;
double большое = маленькое;  // автоматически преобразуется в 10.0

// Явное преобразование (сужение)
double десятичное = 3.14;
int целое = (int) десятичное;  // будет 3 (отбрасывается дробная часть)
```

**Операции с разными типами:**
```java
int a = 5;
double b = 2.5;
double результат = a + b;  // результат будет double (7.5)

String текст = "Число: " + 5;  // "Число: 5" (автоматическое преобразование в строку)
```

**Константы (final):**
```java
final int МАКСИМАЛЬНЫЙ_ВОЗРАСТ = 120;  // нельзя изменить
final double PI = 3.14159;
// МАКСИМАЛЬНЫЙ_ВОЗРАСТ = 150;  // ОШИБКА! Константу нельзя изменить
```

**Проверка типов:**
- Java строго проверяет типы на этапе компиляции
- Нельзя присвоить значение одного типа переменной другого типа без явного преобразования
- Это помогает избежать ошибок в программе

## Уровень 3: Условные операторы (if/else)

### Теория

**Условные операторы** позволяют программе принимать решения и выполнять разный код в зависимости от условий. Это основа логики программирования.

**Синтаксис if:**
```java
if (условие) {
    // код выполнится, если условие true
}
```

**Как работает:**
1. Проверяется условие в скобках
2. Если условие `true` - выполняется код в фигурных скобках
3. Если условие `false` - код пропускается

**Синтаксис if-else:**
```java
if (условие) {
    // код выполнится, если условие true
} else {
    // код выполнится, если условие false
}
```

**Синтаксис if-else if-else (множественные условия):**
```java
if (условие1) {
    // код 1
} else if (условие2) {
    // код 2
} else if (условие3) {
    // код 3
} else {
    // код, если ни одно условие не выполнилось
}
```

**Важно:** Проверка идет сверху вниз. Как только найдено первое истинное условие, выполняется его блок и остальные проверки пропускаются.

**Операторы сравнения:**
- `==` - равно (проверка равенства)
- `!=` - не равно
- `>` - больше
- `<` - меньше
- `>=` - больше или равно
- `<=` - меньше или равно

**Внимание:** `==` проверяет равенство, `=` - это присваивание!

**Логические операторы:**
- `&&` - И (логическое И). Оба условия должны быть `true`
- `||` - ИЛИ (логическое ИЛИ). Хотя бы одно условие должно быть `true`
- `!` - НЕ (логическое отрицание). Инвертирует значение

**Таблица истинности для && (И):**
```
true && true = true
true && false = false
false && true = false
false && false = false
```

**Таблица истинности для || (ИЛИ):**
```
true || true = true
true || false = true
false || true = true
false || false = false
```

**Примеры:**
```java
int age = 18;
if (age >= 18) {
    System.out.println("Совершеннолетний");
} else {
    System.out.println("Несовершеннолетний");
}

int number = 7;
if (number % 2 == 0) {  // % - остаток от деления
    System.out.println("Четное");
} else {
    System.out.println("Нечетное");
}

// Множественные условия
int score = 85;
if (score >= 90) {
    System.out.println("Отлично");
} else if (score >= 70) {
    System.out.println("Хорошо");
} else if (score >= 50) {
    System.out.println("Удовлетворительно");
} else {
    System.out.println("Неудовлетворительно");
}

// Сложные условия
int age = 20;
boolean студент = true;
if (age >= 18 && студент) {
    System.out.println("Совершеннолетний студент");
}

int температура = 25;
if (температура < 0 || температура > 40) {
    System.out.println("Экстремальная температура");
}
```

**Проверка четности числа:**
Число четное, если остаток от деления на 2 равен 0: `число % 2 == 0`
- Четное: 2, 4, 6, 8, 10... (остаток от деления на 2 = 0)
- Нечетное: 1, 3, 5, 7, 9... (остаток от деления на 2 = 1)

**Вложенные условия:**
```java
int age = 20;
if (age >= 18) {
    if (age < 65) {
        System.out.println("Взрослый");
    } else {
        System.out.println("Пенсионер");
    }
} else {
    System.out.println("Несовершеннолетний");
}
```

**Тернарный оператор (краткая запись if-else):**
```java
int age = 20;
String статус = (age >= 18) ? "Взрослый" : "Ребенок";
// Если age >= 18, то "Взрослый", иначе "Ребенок"
```

## Уровень 4: Циклы (for, while)

### Теория

**Циклы** позволяют выполнять один и тот же код несколько раз. Это основа автоматизации в программировании.

**Цикл for:**
```java
for (начало; условие; шаг) {
    // код, который повторяется
}
```

**Как работает цикл for:**
1. Выполняется блок "начало" (обычно создание переменной)
2. Проверяется "условие"
3. Если условие `true` - выполняется код в фигурных скобках
4. Выполняется "шаг" (обычно изменение переменной)
5. Возврат к шагу 2

**Пример:**
```java
for (int i = 1; i <= 10; i++) {
    System.out.println(i);  // Выведет числа от 1 до 10
}
```

**Разбор по шагам:**
- `int i = 1` - начальное значение (создаем переменную i = 1, выполняется один раз)
- `i <= 10` - условие (продолжать пока i <= 10, проверяется перед каждой итерацией)
- `i++` - шаг (увеличить i на 1 после каждой итерации)

**Последовательность выполнения:**
```
Итерация 1: i=1, проверка 1<=10 (true), вывод 1, i становится 2
Итерация 2: i=2, проверка 2<=10 (true), вывод 2, i становится 3
...
Итерация 10: i=10, проверка 10<=10 (true), вывод 10, i становится 11
Итерация 11: i=11, проверка 11<=10 (false), цикл завершается
```

**Обратный порядок:**
```java
for (int i = 10; i >= 1; i--) {
    System.out.println(i);  // Выведет числа от 10 до 1
}
```

**С шагом 2:**
```java
for (int i = 0; i <= 10; i += 2) {
    System.out.println(i);  // Выведет: 0, 2, 4, 6, 8, 10
}
```

**Цикл while:**
```java
while (условие) {
    // код, который повторяется
}
```

**Как работает:**
1. Проверяется условие
2. Если `true` - выполняется код
3. Возврат к шагу 1

**Пример:**
```java
int i = 1;
while (i <= 10) {
    System.out.println(i);
    i++;  // ВАЖНО: увеличивать переменную, иначе цикл будет бесконечным!
}
```

**Цикл do-while:**
```java
do {
    // код выполнится хотя бы один раз
} while (условие);
```

**Разница:** В `do-while` код выполняется сначала, потом проверяется условие. Гарантируется минимум одна итерация.

**Операторы для циклов:**
- `i++` - увеличить i на 1 (постфиксный инкремент)
- `++i` - увеличить i на 1 (префиксный инкремент)
- `i--` - уменьшить i на 1
- `i += 2` - увеличить i на 2 (то же что `i = i + 2`)
- `i -= 3` - уменьшить i на 3
- `i *= 2` - умножить i на 2
- `break` - немедленно выйти из цикла
- `continue` - пропустить оставшийся код итерации, перейти к следующей

**Пример с break:**
```java
for (int i = 1; i <= 10; i++) {
    if (i == 5) {
        break;  // выйти из цикла, когда i станет 5
    }
    System.out.println(i);  // Выведет: 1, 2, 3, 4
}
```

**Пример с continue:**
```java
for (int i = 1; i <= 10; i++) {
    if (i % 2 == 0) {
        continue;  // пропустить четные числа
    }
    System.out.println(i);  // Выведет только нечетные: 1, 3, 5, 7, 9
}
```

**Накопление суммы:**
```java
int сумма = 0;
for (int i = 1; i <= 100; i++) {
    сумма += i;  // то же что: сумма = сумма + i
}
System.out.println("Сумма: " + сумма);  // 5050
```

**Факториал числа:**
Факториал n (n!) = n × (n-1) × (n-2) × ... × 2 × 1
Например: 
- 5! = 5 × 4 × 3 × 2 × 1 = 120
- 0! = 1 (по определению)

```java
int n = 5;
int факториал = 1;
for (int i = 1; i <= n; i++) {
    факториал *= i;  // факториал = факториал * i
}
System.out.println(факториал);  // 120
```

**Бесконечный цикл (осторожно!):**
```java
while (true) {
    // код будет выполняться бесконечно
    // нужен break для выхода
}
```

**Вложенные циклы:**
```java
for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 3; j++) {
        System.out.println(i + " " + j);
    }
}
// Выведет:
// 1 1, 1 2, 1 3
// 2 1, 2 2, 2 3
// 3 1, 3 2, 3 3
```

## Уровень 5: Вложенные циклы

### Теория

**Вложенные циклы** - это циклы внутри других циклов. Внешний цикл выполняется полностью для каждой итерации внутреннего цикла.

**Структура:**
```java
for (int i = 0; i < 5; i++) {        // внешний цикл
    for (int j = 0; j < 5; j++) {    // внутренний цикл
        System.out.print("*");
    }
    System.out.println();  // переход на новую строку
}
```

**Результат:**
```
*****
*****
*****
*****
*****
```

**Как это работает:**
1. Внешний цикл начинает с i = 0
2. Внутренний цикл выполняется полностью (j от 0 до 4) - выводит 5 звездочек
3. Переход на новую строку
4. Внешний цикл переходит к i = 1
5. Снова внутренний цикл выполняется полностью
6. И так далее...

**Треугольник:**
```java
for (int i = 1; i <= 5; i++) {
    for (int j = 1; j <= i; j++) {  // количество звездочек = номер строки
        System.out.print("*");
    }
    System.out.println();
}
```

**Результат:**
```
*
**
***
****
*****
```

**Важно:**
- `System.out.print()` - выводит без перехода на новую строку
- `System.out.println()` - выводит и переходит на новую строку

## Уровень 6: Массивы

### Теория

**Массив** - это структура данных для хранения фиксированного количества значений одного типа в последовательных ячейках памяти. Все элементы массива имеют один тип.

**Создание массива:**

**Способ 1: с указанием размера**
```java
int[] числа = new int[5];  // массив из 5 элементов
// Все элементы инициализируются значениями по умолчанию:
// для int - 0, для double - 0.0, для boolean - false, для объектов - null
```

**Способ 2: с начальными значениями**
```java
int[] числа = {1, 2, 3, 4, 5};
// Размер определяется автоматически по количеству элементов
```

**Способ 3: объявление и инициализация отдельно**
```java
int[] числа;
числа = new int[5];
// или
числа = new int[]{1, 2, 3, 4, 5};
```

**Доступ к элементам:**
- Индексация начинается с **0** (не с 1!)
- Первый элемент: `массив[0]`
- Последний элемент: `массив[размер - 1]`
- Попытка доступа к несуществующему индексу вызовет ошибку `ArrayIndexOutOfBoundsException`

**Пример:**
```java
int[] числа = {10, 20, 30, 40, 50};
System.out.println(числа[0]);  // выведет 10 (первый элемент)
System.out.println(числа[4]);  // выведет 50 (последний элемент)
// System.out.println(числа[5]);  // ОШИБКА! Индекс 5 не существует
```

**Изменение элементов:**
```java
int[] числа = {1, 2, 3, 4, 5};
числа[0] = 100;  // теперь массив: {100, 2, 3, 4, 5}
числа[2] = числа[2] * 2;  // теперь массив: {100, 2, 6, 4, 5}
```

**Длина массива:**
```java
int[] числа = {1, 2, 3, 4, 5};
int длина = числа.length;  // вернет 5 (не индекс, а количество!)
// Важно: length - это свойство, не метод (без скобок)
```

**Перебор массива (классический способ):**
```java
int[] числа = {1, 2, 3, 4, 5};
for (int i = 0; i < числа.length; i++) {
    System.out.println("Индекс " + i + ": " + числа[i]);
}
// Выведет:
// Индекс 0: 1
// Индекс 1: 2
// ...
```

**For-each цикл (упрощенный способ):**
```java
int[] числа = {1, 2, 3, 4, 5};
for (int число : числа) {
    System.out.println(число);
}
// Выведет: 1, 2, 3, 4, 5
```

**Разница между for и for-each:**
- `for` - дает доступ к индексу, можно изменять элементы
- `for-each` - только чтение, проще синтаксис, но нет доступа к индексу

**Массивы строк:**
```java
String[] фрукты = {"Яблоко", "Банан", "Апельсин"};
for (String фрукт : фрукты) {
    System.out.println(фрукт);
}

// Доступ по индексу
System.out.println(фрукты[0]);  // "Яблоко"
```

**Поиск максимума:**
```java
int[] числа = {5, 2, 8, 1, 9};
int максимум = числа[0];  // предполагаем, что первый элемент - максимум
for (int i = 1; i < числа.length; i++) {
    if (числа[i] > максимум) {
        максимум = числа[i];  // нашли больший элемент
    }
}
System.out.println("Максимум: " + максимум);  // 9
```

**Поиск минимума:**
```java
int[] числа = {5, 2, 8, 1, 9};
int минимум = числа[0];
for (int i = 1; i < числа.length; i++) {
    if (числа[i] < минимум) {
        минимум = числа[i];
    }
}
```

**Сумма элементов:**
```java
int[] числа = {1, 2, 3, 4, 5};
int сумма = 0;
for (int число : числа) {
    сумма += число;
}
System.out.println("Сумма: " + сумма);  // 15
```

**Среднее арифметическое:**
```java
int[] числа = {1, 2, 3, 4, 5};
int сумма = 0;
for (int число : числа) {
    сумма += число;
}
double среднее = (double) сумма / числа.length;  // важно преобразовать в double
System.out.println("Среднее: " + среднее);  // 3.0
```

**Обратный порядок:**
```java
int[] числа = {1, 2, 3, 4, 5};
for (int i = числа.length - 1; i >= 0; i--) {
    System.out.println(числа[i]);
}
// Выведет: 5, 4, 3, 2, 1
```

**Многомерные массивы (для справки):**
```java
int[][] матрица = {{1, 2, 3}, {4, 5, 6}};  // 2 строки, 3 столбца
System.out.println(матрица[0][1]);  // 2
```

**Важные замечания:**
- Размер массива фиксирован после создания (нельзя изменить)
- Все элементы одного типа
- Индексация с 0
- Массив - это объект в Java

## Уровень 7: Методы (функции)

### Теория

**Метод** - это именованный блок кода, который можно вызывать многократно из разных мест программы. Методы помогают:
- Избежать повторения кода (DRY - Don't Repeat Yourself)
- Организовать код логически
- Упростить отладку и тестирование
- Сделать код более читаемым

**Синтаксис метода:**
```java
модификатор типВозвращаемогоЗначения имяМетода(параметры) {
    // код метода
    return значение;  // если метод возвращает значение
}
```

**Компоненты метода:**
- **Модификатор** (`public static`) - пока используем так
- **Тип возвращаемого значения** (`void`, `int`, `String` и т.д.)
- **Имя метода** - должно быть понятным и описывать действие
- **Параметры** - данные, которые метод принимает (в скобках)
- **Тело метода** - код, который выполняется

**Типы методов:**

**1. Метод без параметров и без возвращаемого значения:**
```java
public static void приветствие() {
    System.out.println("Привет!");
}
```
- `void` означает "ничего не возвращает"
- Скобки `()` пустые - нет параметров

**2. Метод с параметрами:**
```java
public static void вывестиИмя(String имя) {
    System.out.println("Привет, " + имя + "!");
}
```
- `String имя` - параметр типа String с именем "имя"
- Можно передать несколько параметров через запятую

**3. Метод с возвращаемым значением:**
```java
public static int сумма(int a, int b) {
    return a + b;
}
```
- `int` - тип возвращаемого значения
- `return` - возвращает значение и завершает метод

**4. Метод с несколькими параметрами:**
```java
public static void вывестиИнформацию(String имя, int возраст) {
    System.out.println("Имя: " + имя + ", Возраст: " + возраст);
}
```

**Вызов метода:**
```java
приветствие();  // вызов метода без параметров
вывестиИмя("Никита");  // вызов метода с параметром
int результат = сумма(5, 3);  // вызов метода и сохранение результата
вывестиИнформацию("Никита", 20);  // несколько параметров
```

**Ключевые слова:**
- `void` - метод ничего не возвращает (пустота)
- `return` - возвращает значение и немедленно завершает метод
- `public` - метод доступен извне класса
- `static` - метод принадлежит классу, а не объекту (можно вызывать без создания объекта)

**Return в void методах:**
```java
public static void проверка(int число) {
    if (число < 0) {
        return;  // досрочный выход из метода
    }
    System.out.println("Число положительное: " + число);
}
```

**Локальные переменные:**
Переменные, объявленные внутри метода, видны только в этом методе:
```java
public static void пример() {
    int x = 10;  // локальная переменная
    System.out.println(x);
}
// x здесь недоступна!
```

**Передача параметров:**
В Java параметры передаются **по значению** (для примитивных типов):
```java
public static void изменить(int число) {
    число = 100;  // изменяется только локальная копия
}

int x = 5;
изменить(x);
System.out.println(x);  // все еще 5, не 100!
```

**Пример полной программы:**
```java
public class Пример {
    public static void main(String[] args) {
        приветствие();
        int результат = сумма(10, 20);
        System.out.println("Результат: " + результат);  // выведет 30
        
        вывестиИнформацию("Никита", 20);
        
        boolean четное = четноеЛи(7);
        System.out.println("Четное? " + четное);  // false
    }
    
    public static void приветствие() {
        System.out.println("Привет!");
    }
    
    public static int сумма(int a, int b) {
        return a + b;
    }
    
    public static void вывестиИнформацию(String имя, int возраст) {
        System.out.println("Имя: " + имя + ", Возраст: " + возраст);
    }
    
    public static boolean четноеЛи(int число) {
        return число % 2 == 0;
    }
}
```

**Перегрузка методов (для справки):**
Можно создать несколько методов с одним именем, но разными параметрами:
```java
public static int сумма(int a, int b) {
    return a + b;
}

public static int сумма(int a, int b, int c) {
    return a + b + c;
}

// Вызов:
сумма(1, 2);      // вызовется первый метод
сумма(1, 2, 3);   // вызовется второй метод
```

**Рекурсия (для справки):**
Метод может вызывать сам себя:
```java
public static int факториал(int n) {
    if (n <= 1) {
        return 1;
    }
    return n * факториал(n - 1);  // рекурсивный вызов
}
```

## Уровень 8: Строки (String)

### Теория

**String** - это класс для работы с текстом в Java. Строка - это последовательность символов. В Java строки **неизменяемы** (immutable) - после создания строку нельзя изменить, можно только создать новую.

**Создание строки:**
```java
String текст = "Привет, мир!";
String пустая = "";  // пустая строка
String многострочная = "Строка 1\nСтрока 2";  // \n - новая строка
```

**Важно:** String - это объект, не примитивный тип!

**Основные методы String:**

**1. Длина строки:**
```java
String текст = "Java";
int длина = текст.length();  // вернет 4
// Обратите внимание: length() - это метод (со скобками), не свойство!
```

**2. Верхний и нижний регистр:**
```java
String текст = "Java";
String верхний = текст.toUpperCase();  // "JAVA"
String нижний = текст.toLowerCase();  // "java"
// Оригинальная строка не изменяется!
System.out.println(текст);  // все еще "Java"
```

**3. Проверка содержимого:**
```java
String текст = "Привет, мир!";
boolean содержит = текст.contains("мир");  // true
boolean содержит2 = текст.contains("Python");  // false
```

**4. Объединение строк (конкатенация):**
```java
String строка1 = "Привет";
String строка2 = "мир";
String результат = строка1 + " " + строка2;  // "Привет мир"

// Можно объединять несколько строк
String полное = "Привет" + ", " + "мир" + "!";  // "Привет, мир!"

// С числами (автоматическое преобразование)
String текст = "Число: " + 42;  // "Число: 42"
```

**5. Замена:**
```java
String текст = "Java программирование";
String новый = текст.replace("Java", "Python");  // "Python программирование"
String новый2 = текст.replace("а", "о");  // "Jovo прогроммировонie" (заменяет все вхождения)
```

**6. Получение символа:**
```java
String текст = "Hello";
char первый = текст.charAt(0);  // 'H' (индекс 0)
char последний = текст.charAt(текст.length() - 1);  // 'o'
// charAt() возвращает char, не String!
```

**7. Подстрока:**
```java
String текст = "программирование";
String подстрока1 = текст.substring(3);  // "граммирование" (с индекса 3 до конца)
String подстрока2 = текст.substring(3, 7);  // "грам" (с индекса 3 до 6, не включая 7)
// substring(начало, конец) - конец не включается!
```

**8. Удаление пробелов:**
```java
String текст = "  Java  ";
String безПробелов = текст.trim();  // "Java" (убирает пробелы в начале и конце)
String сПробелами = "  Java  ";
// trim() не изменяет оригинал, создает новую строку
```

**9. Разделение строки:**
```java
String текст = "яблоко,банан,апельсин";
String[] части = текст.split(",");  // массив: ["яблоко", "банан", "апельсин"]
// split() возвращает массив строк

String дата = "2024-01-15";
String[] частиДаты = дата.split("-");  // ["2024", "01", "15"]
```

**10. Проверка начала и конца:**
```java
String текст = "Java";
boolean начинается = текст.startsWith("Ja");  // true
boolean заканчивается = текст.endsWith("va");  // true
boolean начинается2 = текст.startsWith("ja");  // false (регистр важен!)
```

**11. Сравнение строк:**
```java
String строка1 = "Java";
String строка2 = "Java";
String строка3 = "java";

boolean равны1 = строка1.equals(строка2);  // true
boolean равны2 = строка1.equals(строка3);  // false (регистр важен)
boolean равны3 = строка1.equalsIgnoreCase(строка3);  // true (игнорирует регистр)

// ВАЖНО: НЕ используйте == для сравнения строк!
// == сравнивает ссылки, а не содержимое
```

**12. Поиск индекса:**
```java
String текст = "Hello World";
int индекс = текст.indexOf("World");  // 6 (индекс начала подстроки)
int индекс2 = текст.indexOf("o");  // 4 (первое вхождение)
int индекс3 = текст.indexOf("o", 5);  // 7 (поиск с индекса 5)
int неНайдено = текст.indexOf("Python");  // -1 (не найдено)
```

**13. Проверка на пустоту:**
```java
String текст = "";
boolean пустая = текст.isEmpty();  // true
boolean пустая2 = "   ".trim().isEmpty();  // true (после удаления пробелов)
```

**14. Форматирование (для справки):**
```java
String имя = "Никита";
int возраст = 20;
String результат = String.format("Имя: %s, Возраст: %d", имя, возраст);
// "Имя: Никита, Возраст: 20"
```

**Важные замечания:**
- Индексы в строках начинаются с **0**
- `substring(3, 7)` берет символы с индекса 3 до 6 (не включая 7)
- Строки неизменяемы - методы создают новые строки, не изменяют существующие
- `==` сравнивает ссылки, `equals()` сравнивает содержимое
- `length()` - метод, не свойство (со скобками!)
- Для сравнения без учета регистра используйте `equalsIgnoreCase()`

**Эффективное объединение строк:**
Для множественного объединения лучше использовать StringBuilder (для справки):
```java
StringBuilder sb = new StringBuilder();
sb.append("Привет");
sb.append(" ");
sb.append("мир");
String результат = sb.toString();  // "Привет мир"
```

## Уровень 9: Классы и объекты

### Теория

**Класс** - это шаблон (чертеж) для создания объектов. Он определяет структуру и поведение объектов. **Объект** - это конкретный экземпляр класса, созданный по этому шаблону.

**Аналогия:** Класс = чертеж дома, Объект = конкретный дом, построенный по чертежу.

**Создание класса:**
```java
public class Человек {
    // Поля (переменные класса) - описывают состояние объекта
    String имя;
    int возраст;
    String город;
    
    // Методы класса - описывают поведение объекта
    public void представиться() {
        System.out.println("Меня зовут " + имя + ", мне " + возраст + " лет");
    }
    
    public void сказатьГород() {
        System.out.println("Я из города " + город);
    }
}
```

**Создание объекта:**
```java
// В методе main или другом методе:
Человек человек1 = new Человек();  // создание объекта
человек1.имя = "Никита";           // установка значений полей
человек1.возраст = 20;
человек1.город = "Москва";
человек1.представиться();          // вызов метода объекта
человек1.сказатьГород();

// Можно создать несколько объектов одного класса
Человек человек2 = new Человек();
человек2.имя = "Мария";
человек2.возраст = 25;
человек2.город = "Санкт-Петербург";
человек2.представиться();
```

**Разбор создания объекта:**
- `Человек` - тип переменной (класс)
- `человек1` - имя переменной (ссылка на объект)
- `new` - ключевое слово для создания нового объекта
- `Человек()` - вызов конструктора (пока используем конструктор по умолчанию)
- `.` - оператор доступа к полям и методам объекта

**Поля класса:**
- Поля описывают характеристики объекта
- Каждый объект имеет свои собственные копии полей
- Доступ к полям: `объект.поле`

**Методы класса:**
- Методы описывают действия, которые может выполнять объект
- Методы могут использовать поля класса
- `void` - метод ничего не возвращает
- Метод может возвращать значение любого типа

**Пример с возвращаемым значением:**
```java
public class Прямоугольник {
    double длина;
    double ширина;
    
    public double площадь() {
        return длина * ширина;
    }
    
    public double периметр() {
        return 2 * (длина + ширина);
    }
    
    public void информация() {
        System.out.println("Длина: " + длина + ", Ширина: " + ширина);
        System.out.println("Площадь: " + площадь());
        System.out.println("Периметр: " + периметр());
    }
}

// Использование:
Прямоугольник прямоугольник = new Прямоугольник();
прямоугольник.длина = 5.0;
прямоугольник.ширина = 3.0;
double площадь = прямоугольник.площадь();  // 15.0
double периметр = прямоугольник.периметр();  // 16.0
прямоугольник.информация();
```

**Методы с параметрами:**
```java
public class Калькулятор {
    public double сложить(double a, double b) {
        return a + b;
    }
    
    public double вычесть(double a, double b) {
        return a - b;
    }
    
    public void вывестиРезультат(double результат) {
        System.out.println("Результат: " + результат);
    }
}

// Использование:
Калькулятор калькулятор = new Калькулятор();
double сумма = калькулятор.сложить(10, 5);
калькулятор.вывестиРезультат(сумма);
```

**Использование this (для справки):**
```java
public class Человек {
    String имя;
    
    public void установитьИмя(String имя) {
        this.имя = имя;  // this.имя - поле класса, имя - параметр
    }
}
```

**Файлы и классы:**
- Обычно каждый класс в отдельном файле: `ИмяКласса.java`
- Имя файла должно совпадать с именем класса
- Можно несколько классов в одном файле, но только один `public`
- `public` класс должен быть в файле с таким же именем

**Важно:**
- `new` - ключевое слово для создания объекта
- Объекты создаются в памяти во время выполнения программы
- Каждый объект независим от других объектов того же класса
- Поля имеют значения по умолчанию: числа = 0, boolean = false, объекты = null

## Уровень 10: Конструкторы

### Теория

**Конструктор** - это специальный метод, который автоматически вызывается при создании объекта с помощью `new`. Он используется для инициализации полей объекта начальными значениями.

**Синтаксис конструктора:**
```java
public class Человек {
    String имя;
    int возраст;
    
    // Конструктор
    public Человек(String имя, int возраст) {
        this.имя = имя;
        this.возраст = возраст;
    }
}
```

**Особенности конструктора:**
- Имя конструктора **точно совпадает** с именем класса
- Не имеет типа возвращаемого значения (даже `void` не пишется!)
- Вызывается автоматически при создании объекта
- Может принимать параметры для инициализации

**Использование конструктора:**
```java
// Старый способ (без конструктора):
Человек человек1 = new Человек();
человек1.имя = "Никита";
человек1.возраст = 20;

// Новый способ (с конструктором):
Человек человек2 = new Человек("Никита", 20);
// Объект создан и сразу инициализирован - удобнее!
```

**Ключевое слово this:**
`this` - это ссылка на текущий объект. Используется, когда имя параметра совпадает с именем поля:
```java
public class Человек {
    String имя;
    int возраст;
    
    public Человек(String имя, int возраст) {
        this.имя = имя;      // this.имя - поле класса
        this.возраст = возраст;  // имя - параметр конструктора
    }
}
```

**Конструктор по умолчанию:**
Если вы не создали конструктор, Java автоматически создает пустой конструктор:
```java
public class Человек {
    String имя;
    int возраст;
    // Java автоматически создаст: public Человек() { }
}
```

**Но:** Если вы создали конструктор с параметрами, конструктор по умолчанию НЕ создается автоматически!

**Несколько конструкторов (перегрузка):**
Можно создать несколько конструкторов с разными параметрами:
```java
public class Человек {
    String имя;
    int возраст;
    String город;
    
    // Конструктор с полными данными
    public Человек(String имя, int возраст, String город) {
        this.имя = имя;
        this.возраст = возраст;
        this.город = город;
    }
    
    // Конструктор без города
    public Человек(String имя, int возраст) {
        this.имя = имя;
        this.возраст = возраст;
        this.город = "Неизвестно";
    }
    
    // Конструктор по умолчанию
    public Человек() {
        this.имя = "Неизвестно";
        this.возраст = 0;
        this.город = "Неизвестно";
    }
}

// Использование:
Человек человек1 = new Человек("Никита", 20, "Москва");
Человек человек2 = new Человек("Мария", 25);
Человек человек3 = new Человек();
```

**Вызов одного конструктора из другого:**
```java
public class Человек {
    String имя;
    int возраст;
    
    public Человек(String имя, int возраст) {
        this.имя = имя;
        this.возраст = возраст;
    }
    
    public Человек(String имя) {
        this(имя, 0);  // вызов другого конструктора
    }
}
```

**Преимущества конструкторов:**
- Гарантируют, что объект создан с правильными начальными значениями
- Упрощают создание объектов
- Делают код более читаемым
- Предотвращают ошибки (нельзя забыть инициализировать поле)

**Пример полного класса:**
```java
public class Книга {
    String название;
    String автор;
    int год;
    
    // Конструктор
    public Книга(String название, String автор, int год) {
        this.название = название;
        this.автор = автор;
        this.год = год;
    }
    
    // Метод
    public void информация() {
        System.out.println(название + " - " + автор + " (" + год + ")");
    }
}

// Использование:
Книга книга = new Книга("Война и мир", "Толстой", 1869);
книга.информация();
```

## Уровень 11: Массивы объектов

### Теория

Можно создавать массивы не только из примитивных типов, но и из объектов.

**Создание массива объектов:**
```java
Человек[] люди = new Человек[5];  // массив из 5 элементов

// Заполнение массива
люди[0] = new Человек("Никита", 20);
люди[1] = new Человек("Мария", 25);
// и т.д.
```

**Перебор массива объектов:**
```java
for (int i = 0; i < люди.length; i++) {
    люди[i].представиться();
}

// Или с for-each:
for (Человек человек : люди) {
    человек.представиться();
}
```

**Поиск в массиве объектов:**
```java
// Найти человека с максимальным возрастом
Человек самыйСтарший = люди[0];
for (int i = 1; i < люди.length; i++) {
    if (люди[i].возраст > самыйСтарший.возраст) {
        самыйСтарший = люди[i];
    }
}
```

**Важно:**
- Сначала создается массив: `new Человек[5]`
- Затем каждый элемент нужно инициализировать: `люди[0] = new Человек(...)`

## Уровень 12: Статические методы и переменные

### Теория

**Статические** (`static`) методы и переменные принадлежат **классу**, а не отдельному объекту. Они существуют в единственном экземпляре для всего класса и доступны без создания объекта.

**Статический метод:**
```java
public class Утилиты {
    public static int максимум(int a, int b) {
        if (a > b) {
            return a;
        } else {
            return b;
        }
    }
    
    public static int минимум(int a, int b) {
        return (a < b) ? a : b;  // тернарный оператор
    }
}

// Вызов без создания объекта:
int результат = Утилиты.максимум(10, 20);  // 20
int результат2 = Утилиты.минимум(10, 20);  // 10
```

**Разница между статическим и обычным методом:**
```java
public class Пример {
    // Обычный метод - нужен объект
    public void обычныйМетод() {
        System.out.println("Обычный метод");
    }
    
    // Статический метод - вызывается через класс
    public static void статическийМетод() {
        System.out.println("Статический метод");
    }
}

// Использование:
Пример пример = new Пример();
пример.обычныйМетод();        // нужен объект

Пример.статическийМетод();    // объект не нужен
```

**Статическая переменная:**
Статическая переменная - одна на весь класс, общая для всех объектов:
```java
public class Счетчик {
    static int количество = 0;  // общая для всех объектов
    
    public Счетчик() {
        количество++;  // увеличивается при создании каждого объекта
        System.out.println("Создано объектов: " + количество);
    }
    
    public static int получитьКоличество() {
        return количество;
    }
}

// Использование:
Счетчик счетчик1 = new Счетчик();  // Создано объектов: 1
Счетчик счетчик2 = new Счетчик();  // Создано объектов: 2
Счетчик счетчик3 = new Счетчик();  // Создано объектов: 3
System.out.println("Всего: " + Счетчик.получитьКоличество());  // Всего: 3
```

**Статические константы:**
Константы обычно делают статическими и `final` (неизменяемыми):
```java
public class Константы {
    public static final double PI = 3.14159;
    public static final double E = 2.71828;
    public static final int МАКСИМАЛЬНЫЙ_ВОЗРАСТ = 120;
}

// Использование:
double площадь = Константы.PI * радиус * радиус;
double окружность = 2 * Константы.PI * радиус;
```

**Правила именования констант:**
- Обычно используют ВЕРХНИЙ_РЕГИСТР с подчеркиваниями
- `final` означает, что значение нельзя изменить после инициализации

**Статические методы в классе с обычными методами:**
```java
public class Математика {
    // Статический метод - утилита
    public static int сумма(int a, int b) {
        return a + b;
    }
    
    // Обычный метод - работает с полями объекта
    private int значение;
    
    public void установитьЗначение(int значение) {
        this.значение = значение;
    }
    
    public int получитьЗначение() {
        return значение;
    }
}

// Использование:
int результат = Математика.сумма(5, 3);  // статический метод

Математика математика = new Математика();
математика.установитьЗначение(10);  // обычный метод
```

**Ограничения статических методов:**
- Статический метод **не может** обращаться к нестатическим полям и методам напрямую
- Статический метод **не может** использовать `this` (нет объекта!)

**Пример ошибки:**
```java
public class Пример {
    int число = 10;  // нестатическое поле
    
    public static void метод() {
        // System.out.println(число);  // ОШИБКА! Нельзя обращаться к нестатическому полю
        // System.out.println(this.число);  // ОШИБКА! this недоступен
    }
}
```

**Когда использовать static:**
- ✅ Для утилитных методов (математические операции, преобразования)
- ✅ Для констант (PI, E, максимальные значения)
- ✅ Для счетчиков, общих для всех объектов
- ✅ Для методов, которые не зависят от состояния объекта

**Когда НЕ использовать static:**
- ❌ Когда метод работает с полями объекта
- ❌ Когда нужен доступ к `this`
- ❌ Когда каждый объект должен иметь свое значение

**Пример использования:**
```java
public class Утилиты {
    // Статические методы-утилиты
    public static int максимум(int a, int b) {
        return (a > b) ? a : b;
    }
    
    public static boolean четное(int число) {
        return число % 2 == 0;
    }
    
    public static int факториал(int n) {
        if (n <= 1) return 1;
        return n * факториал(n - 1);
    }
}

// Использование везде без создания объекта:
int макс = Утилиты.максимум(10, 20);
boolean чет = Утилиты.четное(7);
int факт = Утилиты.факториал(5);
```

## Уровень 13: Наследование

### Теория

**Наследование** - это механизм, позволяющий создавать новый класс на основе существующего. Новый класс (дочерний, наследник) наследует все поля и методы родительского класса (суперкласса) и может добавлять свои или переопределять существующие.

**Синтаксис:**
```java
public class РодительскийКласс {
    // поля и методы
}

public class ДочернийКласс extends РодительскийКласс {
    // дополнительные поля и методы
    // можно переопределять методы родителя
}
```

**Терминология:**
- **Родительский класс** (суперкласс, базовый класс) - класс, от которого наследуются
- **Дочерний класс** (подкласс, производный класс) - класс, который наследуется
- **extends** - ключевое слово для наследования

**Пример:**
```java
// Родительский класс
public class Животное {
    String имя;
    int возраст;
    
    public void звук() {
        System.out.println("Какой-то звук");
    }
    
    public void информация() {
        System.out.println("Имя: " + имя + ", Возраст: " + возраст);
    }
}

// Дочерний класс
public class Собака extends Животное {
    String порода;  // новое поле
    
    // Переопределение метода родителя
    @Override
    public void звук() {
        System.out.println("Гав!");
    }
    
    // Новый метод, которого нет у родителя
    public void принестиМяч() {
        System.out.println(имя + " принес мяч!");
    }
}
```

**Использование:**
```java
Собака собака = new Собака();
собака.имя = "Бобик";        // поле унаследовано от Животное
собака.возраст = 3;          // поле унаследовано от Животное
собака.порода = "Лабрадор";  // собственное поле
собака.звук();               // выведет "Гав!" (переопределенный метод)
собака.информация();         // выведет информацию (унаследованный метод)
собака.принестиМяч();        // собственный метод
```

**Что наследуется:**
- ✅ Все `public` и `protected` поля
- ✅ Все `public` и `protected` методы
- ❌ `private` поля и методы НЕ наследуются напрямую
- ❌ Конструкторы НЕ наследуются (но можно вызвать через `super()`)

**Переопределение методов:**
Дочерний класс может переопределить метод родителя, изменив его поведение:
```java
public class Кошка extends Животное {
    @Override
    public void звук() {
        System.out.println("Мяу!");
    }
}
```

**Аннотация @Override:**
- Необязательна, но **рекомендуется**
- Помогает избежать ошибок (если метод не найден в родителе, будет ошибка компиляции)
- Улучшает читаемость кода

**Множественное наследование:**
В Java класс может наследоваться только от **одного** класса (один родитель):
```java
// Правильно:
public class Собака extends Животное { }

// Неправильно (в Java нельзя):
// public class Собака extends Животное, ДомашнееЖивотное { }
```

**Цепочка наследования:**
```java
public class Животное { }
public class Млекопитающее extends Животное { }
public class Собака extends Млекопитающее { }
// Собака наследует от Млекопитающее, которое наследует от Животное
```

**Ключевое слово super:**
`super` - ссылка на родительский класс:
```java
public class Собака extends Животное {
    String порода;
    
    public Собака(String имя, int возраст, String порода) {
        super();  // вызов конструктора родителя (если есть)
        this.порода = порода;
    }
    
    @Override
    public void информация() {
        super.информация();  // вызов метода родителя
        System.out.println("Порода: " + порода);
    }
}
```

**Преимущества наследования:**
- ✅ Переиспользование кода (DRY - Don't Repeat Yourself)
- ✅ Можно добавлять новые методы в дочерний класс
- ✅ Можно переопределять методы родительского класса
- ✅ Создание иерархии классов (организация кода)
- ✅ Полиморфизм (см. следующий уровень)

**Пример иерархии:**
```java
public class Транспорт {
    int скорость;
    public void двигаться() {
        System.out.println("Транспорт движется");
    }
}

public class Автомобиль extends Транспорт {
    String марка;
    @Override
    public void двигаться() {
        System.out.println("Автомобиль едет по дороге");
    }
}

public class Велосипед extends Транспорт {
    int количествоКолес;
    @Override
    public void двигаться() {
        System.out.println("Велосипед едет по тротуару");
    }
}
```

## Уровень 14: Полиморфизм

### Теория

**Полиморфизм** - это способность объектов разных классов обрабатываться через общий интерфейс родительского класса.

**Пример:**
```java
Животное[] животные = new Животное[3];
животные[0] = new Собака();
животные[1] = new Кошка();
животные[2] = new Собака();

for (Животное животное : животные) {
    животное.звук();  // вызовется переопределенный метод каждого класса
}
```

**Как это работает:**
- Массив типа `Животное[]` может содержать объекты любых классов-наследников
- При вызове метода вызывается версия метода из реального класса объекта (не из типа переменной)
- Это называется "позднее связывание" или "динамическая диспетчеризация"

**Преимущества:**
- Единый интерфейс для разных классов
- Легко добавлять новые классы без изменения существующего кода
- Код становится более гибким и расширяемым

## Уровень 15: Абстрактные классы и интерфейсы

### Теория

**Абстрактный класс** - это класс, который нельзя создать напрямую. Он используется как базовый класс для других классов.

**Синтаксис:**
```java
public abstract class Фигура {
    // Обычные поля и методы
    
    // Абстрактный метод (без реализации)
    public abstract double площадь();
}

public class Круг extends Фигура {
    double радиус;
    
    @Override
    public double площадь() {
        return 3.14 * радиус * радиус;
    }
}
```

**Интерфейс** - это контракт, который класс должен выполнить. Класс может реализовать несколько интерфейсов.

**Синтаксис интерфейса:**
```java
public interface Звук {
    void издаватьЗвук();  // метод без реализации
}

public class Собака implements Звук {
    @Override
    public void издаватьЗвук() {
        System.out.println("Гав!");
    }
}
```

**Разница:**
- **Абстрактный класс** - может содержать реализованные методы и поля
- **Интерфейс** - содержит только объявления методов (в старых версиях Java)
- Класс может наследоваться только от одного класса, но реализовать несколько интерфейсов

**Ключевые слова:**
- `abstract` - для абстрактных классов и методов
- `interface` - для интерфейсов
- `implements` - для реализации интерфейса

## Уровень 16: Коллекции (ArrayList)

### Теория

**ArrayList** - это динамический массив, который может изменять свой размер во время выполнения программы. В отличие от обычного массива, ArrayList автоматически увеличивается при добавлении элементов и уменьшается при удалении.

**Импорт:**
```java
import java.util.ArrayList;
```

**Создание ArrayList:**
```java
ArrayList<String> фрукты = new ArrayList<>();  // список строк
ArrayList<Integer> числа = new ArrayList<>();  // список целых чисел
ArrayList<Книга> книги = new ArrayList<>();    // список объектов
```

**Генерики (Generics):**
`<String>`, `<Integer>` - это генерики, указывают тип элементов в списке. Это обеспечивает типобезопасность.

**Основные методы:**

**1. Добавление элементов:**
```java
ArrayList<String> фрукты = new ArrayList<>();
фрукты.add("Яблоко");      // добавить в конец
фрукты.add("Банан");
фрукты.add(0, "Апельсин");  // добавить по индексу (сдвигает остальные)
// Теперь: ["Апельсин", "Яблоко", "Банан"]
```

**2. Получение элемента:**
```java
String фрукт = фрукты.get(0);  // получить элемент по индексу
// Индексация с 0, как в массивах
```

**3. Размер:**
```java
int размер = фрукты.size();  // количество элементов
// Обратите внимание: size() - метод, не свойство!
```

**4. Изменение элемента:**
```java
фрукты.set(0, "Груша");  // заменить элемент по индексу
```

**5. Удаление:**
```java
фрукты.remove(0);           // удалить по индексу
фрукты.remove("Банан");     // удалить по значению (первое вхождение)
фрукты.clear();             // удалить все элементы
```

**6. Проверка наличия:**
```java
boolean есть = фрукты.contains("Яблоко");  // true, если есть
boolean пустой = фрукты.isEmpty();        // true, если список пуст
int индекс = фрукты.indexOf("Банан");     // индекс первого вхождения, или -1
```

**7. Перебор элементов:**

**Способ 1: for-each (рекомендуется):**
```java
for (String фрукт : фрукты) {
    System.out.println(фрукт);
}
```

**Способ 2: обычный for:**
```java
for (int i = 0; i < фрукты.size(); i++) {
    System.out.println(фрукты.get(i));
}
```

**8. Преобразование в массив:**
```java
String[] массив = фрукты.toArray(new String[0]);
```

**ArrayList с числами:**
```java
ArrayList<Integer> числа = new ArrayList<>();
числа.add(1);
числа.add(2);
числа.add(3);

int сумма = 0;
for (int число : числа) {
    сумма += число;
}
System.out.println("Сумма: " + сумма);
```

**ArrayList объектов:**
```java
ArrayList<Книга> книги = new ArrayList<>();
книги.add(new Книга("Война и мир", "Толстой", 1869));
книги.add(new Книга("Преступление и наказание", "Достоевский", 1866));

for (Книга книга : книги) {
    книга.информация();
}
```

**Поиск в ArrayList:**
```java
// Найти книгу по названию
public Книга найтиКнигу(ArrayList<Книга> книги, String название) {
    for (Книга книга : книги) {
        if (книга.название.equals(название)) {
            return книга;
        }
    }
    return null;  // не найдено
}
```

**Сравнение с обычным массивом:**

| Обычный массив | ArrayList |
|----------------|-----------|
| Фиксированный размер | Динамический размер |
| `массив.length` | `список.size()` |
| `массив[0]` | `список.get(0)` |
| `массив[0] = значение` | `список.set(0, значение)` |
| Нет методов добавления/удаления | Много методов |

**Преимущества ArrayList:**
- ✅ Динамический размер (не нужно указывать размер заранее)
- ✅ Много полезных методов
- ✅ Легко добавлять/удалять элементы
- ✅ Автоматическое управление памятью
- ✅ Типобезопасность (благодаря генерикам)

**Недостатки:**
- ❌ Немного медленнее обычного массива
- ❌ Больше использует памяти

**Практические примеры:**
```java
// Создание и заполнение
ArrayList<String> имена = new ArrayList<>();
имена.add("Никита");
имена.add("Мария");
имена.add("Алексей");

// Поиск
if (имена.contains("Никита")) {
    System.out.println("Никита найден!");
}

// Удаление
имена.remove("Мария");

// Размер
System.out.println("Количество имен: " + имена.size());

// Перебор
for (String имя : имена) {
    System.out.println(имя);
}
```

## Уровень 17: Обработка исключений (try-catch)

### Теория

**Исключения** - это ошибки, которые возникают во время выполнения программы. Их нужно обрабатывать, чтобы программа не завершалась аварийно.

**Синтаксис try-catch:**
```java
try {
    // код, который может вызвать ошибку
} catch (ТипИсключения e) {
    // код обработки ошибки
}
```

**Пример деления на ноль:**
```java
try {
    int результат = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("Ошибка: деление на ноль!");
}
```

**Пример выхода за границы массива:**
```java
int[] числа = {1, 2, 3};
try {
    System.out.println(числа[10]);
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("Индекс выходит за границы массива!");
}
```

**Пример преобразования строки в число:**
```java
try {
    int число = Integer.parseInt("abc");
} catch (NumberFormatException e) {
    System.out.println("Не удалось преобразовать строку в число!");
}
```

**Блок finally:**
```java
try {
    // код
} catch (Exception e) {
    // обработка
} finally {
    // этот код выполнится всегда
}
```

**Общий catch:**
```java
try {
    // код
} catch (Exception e) {
    // обработает любое исключение
    System.out.println("Произошла ошибка: " + e.getMessage());
}
```

## Уровень 18: Файлы (File, чтение/запись)

### Теория

Работа с файлами позволяет сохранять данные на диск и читать их обратно.

**Импорты:**
```java
import java.io.FileWriter;
import java.io.FileReader;
import java.io.BufferedReader;
import java.io.IOException;
```

**Запись в файл:**
```java
try {
    FileWriter writer = new FileWriter("файл.txt");
    writer.write("Привет, Java!");
    writer.close();  // важно закрыть файл
} catch (IOException e) {
    System.out.println("Ошибка записи в файл");
}
```

**Чтение из файла:**
```java
try {
    BufferedReader reader = new BufferedReader(new FileReader("файл.txt"));
    String строка = reader.readLine();
    while (строка != null) {
        System.out.println(строка);
        строка = reader.readLine();
    }
    reader.close();
} catch (IOException e) {
    System.out.println("Ошибка чтения файла");
}
```

**Запись нескольких строк:**
```java
FileWriter writer = new FileWriter("числа.txt");
for (int i = 1; i <= 5; i++) {
    writer.write(i + "\n");  // \n - переход на новую строку
}
writer.close();
```

**Важно:**
- Всегда используйте try-catch при работе с файлами
- Всегда закрывайте файлы (метод `close()`)
- `\n` - символ новой строки
